Конфигурация и переменные окружения

Файл .env — единственный авторитетный источник runtime-параметров. Приоритет актуальных ключей в .env.example и кода в bot/ выше документации.

Обязательные переменные

Переменная Назначение
TELEGRAM_BOT_TOKEN Токен бота от @BotFather
OPENAI_API_KEY Ключ OpenAI-совместимого API
LLMGATEWAY_BASE_URL (или OPENAI_API_BASE) Базовый URL LLM Gateway

Ключевые секции конфигурации

Telegram Core: ADMIN_USER_IDS, ALLOWED_TELEGRAM_USER_IDS, GROUP_TRIGGER_KEYWORD, TELEGRAM_DOWNLOAD_DIR, rate limits, TELEGRAM_LOCAL_MODE (true → localhost:8081).

LLG Gateway / Маршрутизация: LLMGATEWAY_BASE_URL, LLMGATEWAY_HIGH_MODEL, LLMGATEWAY_LIGHT_MODEL, LLMGATEWAY_BIG_CONTEXT_MODEL, OPENAI_BASE_URL.

Мультимодальность: ENABLE_IMAGE_GENERATION/IMAGE_MODEL/IMAGE_SIZE/IMAGE_QUALITY, ENABLE_VISION/VISION_MODEL/VISION_DETAIL, ENABLE_TRANSCRIPTION/TRANSCRIPTION_MODEL/WHISPER_MODEL, ENABLE_TTS/TTS_MODEL/TTS_VOICE/TTS_SPEED.

Сессии: MAX_HISTORY_SIZE, MAX_CONVERSATION_AGE_MINUTES, MAX_SESSIONS (по умолчанию 5, с автоочисткой старейшей), AUTO_CHAT_MODES.

Бюджеты: BUDGET_PERIOD (monthly/daily/weekly/total), USER_BUDGETS (* = безлимит), GUEST_BUDGET, TOKEN_PRICE, BUDGET_ALERT_THRESHOLD. Устаревшие: MONTHLY_USER_BUDGETS → USER_BUDGETS + BUDGET_PERIOD=monthly; MONTHLY_GUEST_BUDGET → GUEST_BUDGET + BUDGET_PERIOD=monthly (fallback с предупреждением).

Плагины: PLUGINS (CSV allow-list), PLUGINS_DIR, PLUGIN_STRICT_VALIDATION, PLUGIN_STORAGE_ROOT (data/), PLUGIN_MENU_PAGE_SIZE (8), SHOW_PLUGINS_USED.

Agent Tools / Субагенты: AGENT_ASK_USER_TIMEOUT_SECONDS (1800); жёстко заданы: MAX_SUBAGENTS (5), MIN_SUBAGENT_TOOL_ROUNDS (10), MAX_SUBAGENT_TOOL_ROUNDS (50), TASKS_TTL_SECONDS (2 дня), DELIVERY_DEDUP_WINDOW_SECONDS (60), DELIVERY_MAX_ARTIFACT_BYTES (49 MiB).

Skills: SKILLS_DIR/SKILLS_WORKDIR, SKILLS_ALLOW_EXECUTION/SKILLS_ALLOW_SCRIPTS (опасно — права процесса), SKILLS_ALLOWED_OPERATORS/SKILLS_SCRIPT_ADMIN_USER_IDS, таймауты установки (180) и скриптов (120), SKILLS_SCRIPT_OUTPUT_MAX_CHARS (12000), SKILLS_SCRIPT_INTERIM_AFTER_SECONDS (20).

MCP: MCP_SERVERS (JSON-конфиг), DEFAULT_MCP_SERVERS, MCP_SERVERS_ALLOWED_USERS, MCP_REQUEST_TIMEOUT.

Hindsight: HINDSIGHT_ENABLED, HINDSIGHT_RECALL_ON_START, HINDSIGHT_SAVE_ON_DELETE, HINDSIGHT_API_URL/HINDSIGHT_BASE_URL, HINDSIGHT_API_TOKEN, HINDSIGHT_AUTO_RECALL/AUTO_SAVE, HINDSIGHT_MEMORY_TYPES, HINDSIGHT_MAX_AUTO_SAVE_ITEMS.

Database: DB_PATH (bot/user_data.db), SQLITE_TIMEOUT, SQLITE_JOURNAL_MODE (WAL), SQLITE_BUSY_TIMEOUT_MS.

Диагностика: LOG_LEVEL, ENABLE_DEBUG_LOGGING, ENABLE_METRICS/METRICS_PORT.

Архитектура runtime

Последовательность инициализации

PluginManager → Database → OpenAIHelper → ChatGPTTelegramBot

Поток обработки запроса

Компонент Ответственность
telegram_bot.py Обработка обновлений Telegram, сессионное управление, inline-режим
openai_helper.py API-вызовы, маршрутизация моделей, vision, TTS, транскрипция, токен-менеджмент
openai_tool_handler.py Выполнение tool calls: парсинг, валидация, маршрутизация, дедупликация, доставка
plugin_manager.py Загрузка плагинов, валидация, namespace-инструменты, инъекция контекста
database.py SQLite: сессии, история, изображения, usage-статистика, WAL
chat_modes_registry.py + chat_modes.yml Режимы чата: системные промпты, allow-list инструментов, feature-флаги

LLM Gateway — трёхуровневая маршрутизация

Уровень Переменная Назначение
high LLMGATEWAY_HIGH_MODEL Основная чат-модель, сложные задачи
light_model LLMGATEWAY_LIGHT_MODEL Классификация, простые запросы
big_context LLMGATEWAY_BIG_CONTEXT_MODEL Длинные контексты, vision

Режимы чата (bot/chat_modes.yml)

Каждый режим определяет: id, system_prompt, model_profile (маршруты для high/light/big_context/vision/image_generation/image_edit/transcription/tts), plugins_allowlist/plugins_denylist, budget, feature-флаги (vision, audio, web_search, deep_research, agent_runtime, hindsight, skills, mcp), agent_limits, tts_voice, response_format. Валидация перекрёстных ссылок через chat_modes_registry.py; изменения требуют перезапуска.

Система плагинов

Базовый класс и механизмы

  • Базовый класс: bot/plugins/plugin.py
  • Идентификация: plugin_id (уникальный в пределах системы)
  • Namespace-инг функций: function_prefix предотвращает коллизии имён
  • Загрузка: bot/plugins/*.py с учётом allow-list PLUGINS
  • Валидация дубликатов: PLUGIN_STRICT_VALIDATION — строгая (ValueError) или мягкая (приоритет первому)
  • Пакетное выполнение: asyncio.gather для параллельных tool calls
  • Инъекция контекста: chat_id/user_id/message_id через RequestContext в kwargs["request_context"]
  • Прямой ответ: поле direct_result в ответе плагина обходит повторный вызов модели
  • Форматы спецификаций: различие для Google (генеративный API) и прочих моделей (OpenAI Function Calling)

Интеграция с внешними системами

MCP (Model Context Protocol): - Транспорты: HTTP (REST API, SSE) и stdio (дочерние процессы) - Конфигурация: bot/config/mcp_servers.json (автосоздание) - Регистрация: register_mcp_server требует server_name, user_id; HTTP — base_url + опционально api_key; stdio — command, args, опционально env - Управление: регистрация/удаление — только администраторы; просмотр списка — все пользователи - Требования к HTTP-серверу: эндпоинт GET /tools в формате OpenAI Function Calling

Hindsight (долговременная память): - Опциональная, автовызов/автосохранение при удалении сессии - Банк памяти: telegram-{user_id} - Операции: recall (поиск), retain (сохранение), list (перечисление), stats - Типы памяти: world, experience, profile, session preference

База данных SQLite

Паттерн singleton с thread-local соединениями

Database  # singleton
# thread-local соединения — каждый поток получает свой cursor
# контекстный менеджер для транзакций

Ключевые настройки

Параметр Значение Назначение
journal_mode WAL Write-Ahead Logging для конкурентности
busy_timeout Настраивается Ожидание при блокировке
foreign_keys ON Каскадное удаление

Структура данных

  • conversation_context.context: JSON {"messages": [...]}
  • Подсчёт message_count: только сообщения роли user
  • Ограничение max_sessions: удаление старейшей сессии при превышении
  • Длительные вызовы OpenAI: вынесены за пределы транзакций

Тестирование

Конфигурация

# pytest.ini
[pytest]
asyncio_mode = auto

Запуск

python -m pytest -q tests/
python -m pytest -q bot/tests/

Синтаксическая проверка

python -m py_compile bot/openai_helper.py bot/telegram_bot.py

Стратегия изоляции тестов

  • Динамическая подмена модулей в sys.modules (tiktoken, pydub, markdown2, tenacity, pandas, numpy, matplotlib) через types.ModuleType
  • Временные директории tmp_path для хранилищ плагинов и БД
  • monkeypatch для переменных окружения и методов
  • AsyncMock/MagicMock для асинхронных зависимостей
  • Сброс singleton Database._instance = None между тестами

Docker-развёртывание

Dockerfile

  • База: python:3.9-alpine
  • Системный пакет: ffmpeg (медиа-обработка)
  • Точка входа: python -m bot
  • Оптимизации: --no-cache для apk, --no-cache-dir для pip

docker-compose.yml

services:
  chatgpt-telegram-bot:
    build:
      context: .
      dockerfile: Dockerfile
    volumes:
      - .:/app          # Горячая перезагрузка кода при разработке
    restart: unless-stopped  # Автоперезапуск при сбоях, кроме ручной остановки

Переменная Значение Назначение

PYTHONDONTWRITEBYTECODE 1 Запрет .pyc
PIP_DISABLE_PIP_VERSION_CHECK on Отключение проверки обновлений pip
