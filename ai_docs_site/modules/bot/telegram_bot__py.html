bot/telegram_bot

Источник: bot/telegram_bot.py

Модуль bot реализует основной класс Telegram-бота ChatGPTTelegramBot (алиас TelegramBot), интегрирующий OpenAI API и альтернативные провайдеры LLM с платформой Telegram. Поддерживает мультиязычность, плагиновую архитектуру, генерацию изображений (DALL·E), голосовых сообщений (TTS), транскрипцию аудио (Whisper), интерпретацию изображений (Vision), отслеживание использования токенов и бюджета, управление пользовательскими сессиями через базу данных, буферизацию сообщений с группировкой по времени, потоковую и пакетную генерацию ответов, inline-запросы с кешированием, интерактивные настройки с пагинацией и интеграцию с агентскими инструментами.

Ключевые структуры данных ChatGPTTelegramBot / TelegramBot — Основной класс бота, инкапсулирующий конфигурацию (BotConfig), состояние диалогов, кеши настроек, буферы сообщений и обработчики Telegram. RequestContext — Контекст запроса с идентификаторами чата, пользователя, сообщения и составным request_id для трассировки. UsageTracker — Трекер использования API для подсчёта расхода токенов, изображений, транскрипций и TTS. BusyStatusMessage / BusyStatus — Индикатор активной обработки с автоматическим обновлением статуса. buffer_data — Структура буфера чата: messages (список), processing (bool), timer (asyncio.Task или None). inline_queries_cache — Словарь кешированных inline-запросов с ограничением по размеру и времени жизни. file_context — Словарь с ключами local_path, file_name, mime_type, file_size, описывающий загруженный файл для передачи в модель. _UpdateProxy — Прокси-класс для унифицированного доступа к message при отсутствии update.message. UserSession — Данные пользовательской сессии, включая историю сообщений и настройки. BackgroundTask — Фоновая задача asyncio, накопленная в _background_tasks для последующей отменки.

Аргументы config — Словарь с конфигурацией бота (BotConfig). openai — Экземпляр OpenAIHelper для взаимодействия с LLM. db — Экземпляр Database для хранения настроек и статистики.

Возвращает Строку file_id последнего (наибольшего) изображения из photo, или file_id документа с MIME-типом image/*, или None, если изображение отсутствует.

_safe_reply_file_name(document) -> str Формирует безопасное имя файла для загруженного документа. Аргументы document — Объект документа Telegram (Document), содержащий метаданные файла. Возвращает Очищенную строку имени файла, пригодную для использования в файловой системе, с заменой небезопасных символов на подчёркивание.

_download_replied_file_for_model(self, update: Update, context: ContextTypes.DEFAULT_TYPE) -> dict | None Загружает файл из ответного сообщения во временную директорию для передачи модели. Аргументы update — Объект обновления Telegram (Update), содержащий сообщение с reply. context — Контекст обработчика Telegram (ContextTypes.DEFAULT_TYPE), предоставляющий доступ к боту. Возвращает Словарь file_context с информацией о загруженном файле, или None, если файл отсутствует или не является документом. Исключения Любое исключение при загрузке файла пробрасывается выше после попытки очистки временных ресурсов.

Аргументы prompt — Исходный текстовый промпт пользователя. file_context — Словарь с метаданными загруженного файла (local_path, file_name, mime_type, file_size). Возвращает Строку промпта с добавленным блоком контекста файла для модели.

_cleanup_replied_file_context(file_context: dict | None) -> None Удаляет временный файл и директорию из контекста ответного сообщения. Аргументы file_context — Словарь с local_path загруженного файла, или None.

_reply_context_kind(self, update: Update) -> str | None Определяет тип контекста ответного сообщения (изображение, текст или другое). Аргументы update — Объект обновления Telegram (Update). Возвращает Строку "image", "text" или "other", или None, если ответное сообщение отсутствует.

_classify_reply_intent(self, update: Update, prompt: str | None) -> str | None Классифицирует намерение пользователя при ответе на сообщение с помощью лёгкой модели. Аргументы update — Объект обновления Telegram (Update). prompt — Текстовый промпт пользователя для классификации. Возвращает Строку с классифицированным намерением ("image_edit", "image_describe", "text_reply") или None при ошибке или отсутствии классификации.

_remember_sent_image_messages(self, update: Update, sent_messages) -> None Сохраняет идентификаторы отправленных изображений в базу данных и обновляет контекст OpenAI. Аргументы update — Объект обновления Telegram (Update), содержащий информацию о пользователе и чате. sent_messages — Отправленное сообщение или список сообщений для анализа.

_get_agent_tools_plugin(self) Получает экземпляр плагина агентских инструментов через менеджер плагинов. Возвращает Объект плагина "agent_tools" или None, если плагин недоступен.

_clear_agent_plan(self, chat_id, user_id, *, reason: str, terminal_only: bool = False) -> None Очищает план задач агента для указанного чата и пользователя. Аргументы chat_id — Идентификатор чата. user_id — Идентификатор пользователя. reason — Причина очистки плана (обязательный именованный аргумент). terminal_only — Флаг очистки только терминальных задач; по умолчанию False.

_build_plan_status_provider(self, chat_id, user_id) Создаёт провайдер статуса плана агента для отображения прогресса выполнения. Аргументы chat_id — Идентификатор чата. user_id — Идентификатор пользователя. Аргументы user_id — идентификатор пользователя Telegram. max_sessions — максимальное количество сессий; более старые подлежат удалению. Возвращает None. Аргументы limit — максимальное количество задач для обработки за один вызов; значение по умолчанию из константы HINDSIGHT_FINALIZE_JOB_LIMIT. Возвращает Количество обработанных задач.
