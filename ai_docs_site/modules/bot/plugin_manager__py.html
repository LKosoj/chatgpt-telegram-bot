bot/plugin_manager

Источник: bot/plugin_manager.py

Менеджер плагинов, отвечающий за динамическую загрузку Python-модулей, их регистрацию, инициализацию с зависимостями, формирование спецификаций функций для LLM-моделей и асинхронное выполнение вызовов с валидацией аргументов. Обеспечивает разрешение имён функций по префиксам, фильтрацию разрешённых плагинов, нормализацию спецификаций инструментов и управление жизненным циклом экземпляров с кешированием. Поддерживает сценарии делегирования функций субагентам с явными списками блокировки.

Ключевые структуры данных Plugin — базовый класс плагина, от которого наследуются все загружаемые плагины; предоставляет методы get_spec, initialize, close и свойства plugin_id/function_prefix PluginSpec — спецификация плагина, возвращаемая методом get_spec() плагина CommandDict — нормализованный словарь команды с полями command, description, handler, handler_kwargs, add_to_menu, plugin_name MenuEntry — словарь с полями command и description для формирования меню бота MessageHandlerDict — словарь обработчика сообщения с опциональным полем plugin_name

class PluginManager Менеджер жизненного цикла плагинов: загрузка, регистрация, инициализация, выполнение и формирование спецификаций функций для LLM. Поля plugins — словарь зарегистрированных классов плагинов (имя → класс) plugin_instances — кэш созданных экземпляров плагинов (имя → объект) openai — ссылка на объект OpenAI для передачи плагинам bot — ссылка на бота, извлекаемая из openai enabled_plugins — список имён активных плагинов из конфигурации strict_validation — флаг строгой валидации (из переменной окружения PLUGIN_STRICT_VALIDATION) storage_root — корневой каталог для хранения данных плагинов plugins_directory — путь к директории с файлами плагинов

init(self, config, plugins_directory="plugins") Конструктор менеджера плагинов Аргументы config — конфигурация с ключом plugins (список имён) plugins_directory — относительный путь к директории плагинов

set_openai(self, openai) Устанавливает ссылку на openai и переинициализирует все существующие экземпляры плагинов Аргументы openai — объект OpenAI для передачи плагинам

load_plugins(self) Загружает все плагины из указанной директории с фильтрацией по enabled_plugins

load_plugin_module(self, plugin_name) Загружает модуль плагина по имени файла Аргументы plugin_name — имя файла плагина без расширения .py Возвращает загруженный модуль или None при ошибке

register_plugin(self, plugin_name, plugin_module) Регистрирует первый найденный класс-наследник Plugin из модуля Аргументы plugin_name — имя плагина plugin_module — загруженный модуль

execute(self, plugin_name, user_id, *args) Асинхронный единый интерфейс для вызова метода execute плагина Аргументы plugin_name — имя вызываемого плагина user_id — идентификатор пользователя для создания/получения экземпляра args — позиционные аргументы для execute Возвращает результат выполнения или сообщение об ошибке

reinitialize(self) Полная перезагрузка всех плагинов с очисткой состояния

get_functions_specs(self, helper, model_to_use, allowed_plugins=None) Формирует спецификации функций для передачи в LLM с учётом модели и фильтра Аргументы helper — экземпляр OpenAIHelper model_to_use — имя используемой модели allowed_plugins — список разрешённых имён, ['All'] или ['None'] Возвращает список спецификаций или объект function_declarations для Google-моделей Исключения ValueError — при обнаружении дубликатов имён функций в strict-режиме

call_function(self, function_name, helper, arguments, request_context=None) Асинхронный вызов функции плагина по имени с валидацией аргументов Аргументы function_name — полное имя функции (включая префикс плагина) helper — экземпляр OpenAIHelper arguments — JSON-строка с аргументами функции request_context — опциональный контекст запроса для передачи в плагин Возвращает JSON-строка с результатом или объектом ошибки Исключения json.JSONDecodeError — при невалидном JSON в arguments

get_plugin_source_name(self, function_name) -> str Возвращает имя источника плагина, предоставляющего указанную функцию. Аргументы function_name — имя функции для поиска Возвращает Строка с именем источника плагина или пустая строка, если плагин не найден

get_spec_by_function_name(self, function_name) Возвращает нормализованную спецификацию функции по её имени. Аргументы function_name — имя функции для поиска Возвращает Словарь спецификации функции или None, если функция не найдена

get_plugin_name_by_function_name(self, function_name) Определяет имя плагина, содержащего функцию с указанным именем. Аргументы function_name — имя функции для разрешения Возвращает Строка с именем плагина или None, если функция не принадлежит ни одному плагину

is_function_allowed(self, function_name, allowed_plugins) Проверяет, разрешён ли вызов функции в контексте заданного списка плагинов. Аргументы function_name — имя проверяемой функции allowed_plugins — список разрешённых плагинов; ['All'] разрешает все, ['None'] или пустой список запрещает все Возвращает True, если функция разрешена; False в противном случае

get_subagent_function_specs(self, helper, model_to_use, parent_allowed_plugins=None, blocked_function_names=None) Формирует спецификации инструментов, доступных субагенту, с учётом родительского списка разрешений и явно заблокированных функций. Аргументы helper — вспомогательный объект для формирования спецификаций model_to_use — идентификатор модели для генерации спецификаций parent_allowed_plugins — родительский список разрешённых плагинов (по умолчанию ['All']) blocked_function_names — множество или список имён заблокированных функций Возвращает Кортеж (filtered_specs, allowed_function_names), где filtered_specs — список OpenAI-стил спецификаций, allowed_function_names — множество разрешённых имён функций

is_subagent_function_allowed(self, function_name, parent_allowed_plugins=None, blocked_function_names=None) Проверяет разрешённость функции для субагента с учётом блок-листа и родительских разрешений. Аргументы function_name — имя проверяемой функции parent_allowed_plugins — родительский список разрешённых плагинов (по умолчанию ['All']) blocked_function_names — множество или список заблокированных имён функций Возвращает True, если функция разрешена для субагента; False в противном случае

__get_plugin_by_function_name(self, function_name) Внутренний метод для получения экземпляра плагина по имени функции. Аргументы function_name — имя функции для поиска Возвращает Экземпляр плагина или None, если плагин не найден

get_plugin(self, plugin_name) Возвращает или создаёт экземпляр плагина по имени с инициализацией и кешированием. Аргументы plugin_name — имя плагина для получения Возвращает Инициализированный экземпляр плагина или None, если плагин не зарегистрирован

close_all(self) Закрывает все кешированные экземпляры плагинов с подавлением ошибок.

_normalize_specs(self, specs: List[Dict], plugin_instance: Plugin) -> List[Dict] Нормализует спецификации функций плагина, добавляя префикс к именам без точки. Аргументы specs — исходный список спецификаций функций plugin_instance — экземпляр плагина для получения префикса Возвращает Список нормализованных спецификаций с полными именами функций

_validate_enabled_plugins(self) Валидирует список явно включённых плагинов, выявляя отсутствующие и незагруженные плагины. Исключения ValueError — при strict_validation=True и наличии проблем с включёнными плагинами

_get_available_plugin_files(self) -> list[str] Возвращает список имён файлов плагинов, доступных в директории плагинов. Возвращает Список строковых имён файлов без расширений

_suggest_plugin_names(self, name: str, candidates: list[str]) -> list[str] Предлагает близкие по написанию имена плагинов из доступных кандидатов. Аргументы name — искомое имя плагина candidates — список доступных имён плагинов для сопоставления Возвращает Список до трёх наиболее похожих имён с коэффициентом схожести ≥ 0.6

filter_allowed_plugins(self, allowed_plugins: List[str] | None) -> List[str] Фильтрует и валидирует список разрешённых плагинов, исключая несуществующие. Аргументы allowed_plugins — список разрешённых плагинов или None Возвращает Отфильтрованный список существующих плагинов или ['All'] Исключения ValueError — при strict_validation=True и наличии несуществующих плагинов

get_all_plugin_descriptions(self) -> list[str] Собирает описания всех функций из всех зарегистрированных плагинов. Возвращает Список словарей с ключами plugin, function, description

get_plugin_spec(self, plugin_name: str) -> List[Dict] Возвращает нормализованные спецификации всех функций указанного плагина. Аргументы plugin_name — имя плагина Возвращает Список спецификаций функций или None, если плагин не найден

has_plugin(self, plugin_name: str) -> bool Проверяет существование плагина по имени. Аргументы plugin_name — имя плагина для проверки Возвращает True, если плагин зарегистрирован; иначе False

get_plugin_commands(self) -> List[Dict] Возвращает список всех команд от всех плагинов с базовой валидацией. Возвращает Список нормализованных словарей команд, прошедших валидацию и проверку на дубликаты

build_bot_commands(self) -> Dict[str, List[Dict]] Формирует регистрации команд плагинов и записи меню. Возвращает Словарь с ключами plugin_commands (валидированные команды для обработчиков) и menu_entries (список команд с описаниями для меню)

_validate_and_normalize_command(self, cmd: Dict, plugin_name: str) -> Dict | None Валидирует и нормализует определение команды от плагина. Аргументы cmd — словарь с определением команды от плагина plugin_name — имя плагина-источника для логирования ошибок Возвращает Нормализованный словарь команды или None при невалидном определении

get_message_handlers(self) -> List[Dict] Возвращает список обработчиков сообщений от всех плагинов. Возвращает Список словарей обработчиков сообщений с заполненным полем plugin_name
