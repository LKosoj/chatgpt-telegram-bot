bot/openai_tool_handler

Источник: bot/openai_tool_handler.py

Модуль обработки вызовов функций LLM. Реализует парсинг tool calls из потоковых и непотоковых ответов, валидацию аргументов, маршрутизацию к плагинам, управление историей диалога, механизм принудительного выполнения обязательных инструментов (delivery contract), а также обработку прямых результатов: отложенную доставку, компактизацию ответов для контекста LLM, слияние результатов и принудительную отправку финального ответа через agent_tools.deliver_to_user в режиме skills_agent.

Ключевые структуры данных tool_call_parts — словарь {index: {"id": str, "name": str, "arguments": str}} для аккумуляции фрагментов tool call при потоковой передаче DELIVERY_TOOL_NAME — Имя инструмента доставки результатов пользователю. DELIVERY_REPAIR_PROMPT — Системный промпт для исправления нарушения протокола доставки. _TEXT_PREVIEW_LIMIT — Максимальная длина текстового превью для компактного представления отложенного результата.

async def _prepend_stream_item(first_item, response) Добавляет первый элемент в начало асинхронного потока. Аргументы first_item — Элемент для вставки в начало потока. response — Исходный асинхронный итератор. Возвращает Асинхронный генератор, сначала выдающий first_item, затем элементы response.

def _direct_result_payload(response) -> dict | None Извлекает полезную нагрузку direct_result из ответа инструмента. Аргументы response — Ответ инструмента (dict, строка JSON или другой тип). Возвращает Словарь direct_result или None, если извлечение невозможно.

def _should_defer_direct_result(response, defer_direct_results: bool) -> bool Определяет, следует ли отложить обработку прямого результата. Аргументы response — Ответ инструмента для проверки. defer_direct_results — Флаг глобального разрешения отложенной доставки. Возвращает True, если результат следует отложить; иначе False.

def _compact_deferred_tool_response(response) -> str Создаёт компактное JSON-представление отложенного direct_result для сохранения в истории чата. Аргументы response — Исходный ответ инструмента с direct_result. Возвращает JSON-строку с сокращённой информацией: метаданные, длина текста, превью, пути артефактов.

def _merge_direct_results_into_final(direct_results: list) -> dict Объединяет несколько direct_result в единый финальный результат. Аргументы direct_results — Список ответов инструментов для слияния. Возвращает Словарь с объединённым direct_result типа "final", содержащим текст и артефакты.

def _delivery_contract_required(helper, chat_id, final_delivery_required: bool) -> bool Проверяет необходимость принудительной доставки через DELIVERY_TOOL_NAME. Аргументы helper — Экземпляр помощника с доступом к conversations и режимам. chat_id — Идентификатор чата. final_delivery_required — Флаг требования финальной доставки. Возвращает True, если текущий режим требует вызова инструмента доставки.

def _tool_response_succeeded(response) -> bool Проверяет успешность выполнения инструмента по его ответу. Аргументы response — Ответ инструмента для проверки. Возвращает False, если ответ содержит ошибку или success=False; иначе True.

def _delivery_tool_is_allowed(helper, allowed_plugins) -> bool Проверяет разрешение использования инструмента доставки. Аргументы helper — Экземпляр помощника с plugin_manager. allowed_plugins — Список/множество разрешённых плагинов. Возвращает True, если инструмент доставки разрешён для использования.

def _delivery_tool_choice(tools) Формирует принудительный выбор инструмента доставки при наличии в списке tools. Аргументы tools — Список доступных инструментов. Возвращает Словарь с принудительным выбором функции или "auto", если инструмент не найден.

def _delivery_contract_error() -> dict Формирует сообщение об ошибке невыполнения контракта доставки. Возвращает Словарь direct_result с текстовым описанием ошибки на русском языке.

async def _retry_missing_delivery_tool(helper, chat_id, stream, times, tools_used, allowed_plugins, user_id, request_context) Повторяет запрос при нарушении протокола доставки: модель вернула текст вместо вызова DELIVERY_TOOL_NAME. Аргументы helper — Экземпляр помощника. chat_id — Идентификатор чата. stream — Флаг потоковой передачи ответа. times — Счётчик попыток (для рекурсии/цикла). tools_used — Список уже использованных инструментов. allowed_plugins — Разрешённые плагины. user_id — Идентификатор пользователя. request_context — Контекст текущего запроса. Возвращает Кортеж (result, tools_used) — результат исправления или ошибка доставки и обновлённый список инструментов. Исключения Может возбуждать исключения при сетевых ошибках или ошибках модели при повторном запросе.

async def handle_function_call(helper, chat_id, response, stream=False, times=0, tools_used=(), allowed_plugins=None, user_id=None, request_context=None, delivery_retry_attempted=False, final_delivery_required=False) Главная функция обработки вызовов функций из ответа LLM. Аргументы helper — экземпляр помощника бота chat_id — идентификатор чата response — ответ от LLM (объект chat.completions) stream — флаг потоковой передачи ответа times — счётчик глубины рекурсивных вызовов tools_used — кортеж уже использованных инструментов allowed_plugins — список разрешённых плагинов (None = ['All']) user_id — идентификатор пользователя request_context — контекст текущего запроса с полями chat_id, user_id, plugin_chat_id, message_id delivery_retry_attempted — флаг попытки повторного вызова delivery contract final_delivery_required — флаг необходимости финального вызова доставки Возвращает Кортеж (response_or_item, tools_used) — либо готовый ответ, либо потоковый элемент с результатами Исключения openai.APIError — при ошибках API в потоковом режиме (перехватывается внутри, логируется)

add_tool_result(tool_name, tool_response, tool_call_id) Добавляет результат вызова инструмента в контекст разговора. Аргументы tool_name — имя вызванного инструмента tool_response — результат выполнения инструмента (строка или сериализованный JSON) tool_call_id — идентификатор вызова инструмента для сопоставления с запросом модели
