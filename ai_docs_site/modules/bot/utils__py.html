bot/utils

Источник: bot/utils.py

Модуль предоставляет вспомогательные функции и классы для Telegram-бота. Включает утилиты для обработки сообщений, проверки членства пользователей в группах, управления статусными сообщениями о прогрессе длительных операций с поддержкой отображения плана выполнения, разбиения длинного текста на фрагменты с сохранением Markdown-форматирования, обёртки асинхронных операций с индикаторами чата, редактирования сообщений с fallback-логикой, обработки ошибок, проверки доступа пользователей, управления бюджетами и отслеживания использования токенов, а также вспомогательные функции для работы с изображениями, медиа-контентом, конвертации Markdown в HTML и отправки длинных ответов в виде файлов.

Ключевые структуры данных BUSY_STATUS_ICONS — словарь соответствия статусов задач плана иконкам-эмодзи: "pending" → "⏳", "in_progress" → "🔄", "completed" → "✅", "cancelled" → "⛔" budget_cost_map — словарь сопоставления периодов бюджета ("monthly", "daily", "all-time") с ключами стоимости в UsageTracker processed_lines — список строк, предварительно разбитых по максимальной длине строки markdown_stack — стек строк, отслеживающий открытые Markdown-элементы (*, , ` и их двойные варианты) chunks — список итоговых чанков текста current_chunk — накапливаемый текущий чанк текста

message_text(message: Message) -> str Возвращает текст сообщения, исключая команды бота. Аргументы message — объект сообщения Telegram Возвращает Очищенный текст сообщения без команд бота или пустую строку

async def is_user_in_group(update: Update, context: CallbackContext, user_id: int) -> bool Проверяет, является ли указанный пользователь членом группы. Аргументы update — объект обновления Telegram context — контекст обратного вызова user_id — идентификатор проверяемого пользователя Возвращает True, если пользователь является владельцем, администратором или обычным членом группы; иначе False Исключения telegram.error.BadRequest — при ошибках API Telegram (кроме "User not found", которое трактуется как False) Exception — при прочих ошибках выполнения

def get_thread_id(update: Update) -> int | None Получает идентификатор треда сообщения для обновления, если применимо. Аргументы update — объект обновления Telegram Возвращает Идентификатор треда (message_thread_id) для топиковых сообщений или None

class BusyStatusMessage Поддерживает временное статусное сообщение для длительных операций в чате. При наличии plan_provider отображает текущие шаги плана с их статусами. Поля update — объект обновления Telegram context — контекст обратного вызова description — описание выполняемой операции config — конфигурация бота (опционально) interval — интервал обновления сообщения в секундах (по умолчанию 30.0) plan_provider — вызываемый объект, возвращающий список задач плана (опционально) message — отправленное статусное сообщение или None Методы async def start(self) -> BusyStatusMessage — запускает фоновую задачу периодического обновления статусного сообщения async def stop(self) -> None — останавливает обновление и удаляет статусное сообщение async def _run(self) -> None — внутренний цикл периодического обновления сообщения async def _send(self) -> bool — отправляет начальное статусное сообщение async def _edit(self) -> None — редактирует существующее статусное сообщение def _text(self) -> str — формирует текст статусного сообщения с учётом прошедшего времени и плана def _plan_lines(self) -> list[str] — формирует строки отображения задач плана с иконками статусов

def get_stream_cutoff_values(update: Update, content: str) -> int Определяет пороговые значения для потоковой передачи на основе длины контента и типа чата. Аргументы update — объект обновления Telegram content — текстовый контент для анализа длины Возвращает Пороговое значение в символах для отправки фрагмента потоковой передачи

def is_group_chat(update: Update) -> bool Проверяет, было ли сообщение отправлено из группового чата. Аргументы update — объект обновления Telegram Возвращает True для групп и супергрупп, иначе False

def split_into_chunks(text: str, chunk_size: int = 4096) -> list[str] Разбивает строку на фрагменты заданного размера с сохранением Markdown-форматирования. Аргументы text — текст для разбиения chunk_size — максимальный размер каждого фрагмента (по умолчанию 4096) Возвращает Список фрагментов с сохранённым Markdown-форматированием

async def wrap_with_indicator(update: Update, context: CallbackContext, coroutine, chat_action: constants.ChatAction = "", is_inline=False) Оборачивает корутину с периодической отправкой индикатора активности чата. Аргументы update — объект обновления Telegram context — контекст обратного вызова бота coroutine — вызываемый объект, возвращающий корутину chat_action — тип индикатора активности чата (по умолчанию пустая строка) is_inline — флаг inline-режима, при True не отправляются chat actions Возвращает Результат выполнения переданной корутины Исключения telegram.error.TimedOut — при превышении таймаута 2000 секунд Exception — прочие ошибки, проброшенные из корутины

async def edit_message_with_retry(context: ContextTypes.DEFAULT_TYPE, chat_id: int | None, message_id: str, text: str, markdown: bool = True, is_inline: bool = False) Редактирует сообщение с повторной попыткой при ошибке Markdown-разметки. Аргументы context — контекст бота для выполнения операций chat_id — идентификатор чата (None для inline-сообщений) message_id — идентификатор сообщения для редактирования text — новый текст сообщения markdown — использовать Markdown-разметку (по умолчанию True) is_inline — флаг inline-сообщения Исключения telegram.error.BadRequest — при невозможности отредактировать сообщение Exception — прочие ошибки редактирования

async def error_handler(_: object, context: ContextTypes.DEFAULT_TYPE) -> None Обрабатывает ошибки, возникающие при обработке обновлений в python-telegram-bot. Аргументы _ — неиспользуемый объект (хендлер обновления) context — контекст с информацией об ошибке

async def is_allowed(config, update: Update, context: CallbackContext, is_inline=False) -> bool Проверяет, разрешён ли пользователю доступ к боту. Аргументы config — конфигурация бота с настройками доступа update — объект обновления Telegram context — контекст обратного вызова is_inline — флаг проверки для inline-запроса Возвращает True если пользователь разрешён, иначе False

def is_admin(config, user_id: int, log_no_admin=False) -> bool Проверяет, является ли пользователь администратором бота. Аргументы config — конфигурация бота со списком admin_user_ids user_id — идентификатор пользователя для проверки log_no_admin — логировать отсутствие администраторов Возвращает True если пользователь в списке администраторов, иначе False

def get_user_budget(config, user_id) -> float | None Возвращает бюджет пользователя на основе конфигурации бота. Аргументы config — конфигурация бота с настройками бюджетов user_id — идентификатор пользователя Возвращает Бюджет пользователя как float, float('inf') для администраторов или при '*' в настройках, None при отсутствии пользователя в списке разрешённых

get_remaining_budget(config, usage, update: Update, is_inline=False) -> float Вычисляет оставшийся бюджет пользователя на основе текущего использования. Аргументы config — объект конфигурации бота usage — объект отслеживания использования update — объект обновления Telegram is_inline — флаг inline-запроса Возвращает Оставшийся бюджет пользователя как float

is_within_budget(config, usage, update: Update, is_inline=False) -> bool Проверяет, не исчерпал ли пользователь свой лимит использования. Аргументы config — объект конфигурации бота usage — объект отслеживания использования update — объект обновления Telegram is_inline — флаг inline-запроса Возвращает True если у пользователя положительный остаток бюджета

add_chat_request_to_usage_tracker(usage, config, user_id, used_tokens) -> bool Добавляет запрос чата в трекер использования для пользователя и гостей при необходимости. Аргументы usage — объект отслеживания использования config — объект конфигурации бота user_id — идентификатор пользователя used_tokens — количество использованных токенов Возвращает True при успешном добавлении, False при ошибке или нулевом количестве токенов

get_reply_to_message_id(config, update: Update) -> int | None Возвращает идентификатор сообщения для ответа при включённом цитировании или в групповых чатах. Аргументы config — объект конфигурации бота update — объект обновления Telegram Возвращает Идентификатор сообщения или None если цитирование отключено

compute_scope_key(chat_id=None, user_id=None) -> str Формирует канонический ключ области видимости состояния плагина. Аргументы chat_id — идентификатор чата user_id — идентификатор пользователя Возвращает Строка ключа в формате "chat:", "user:" или "global"

is_direct_result(response: any) -> bool Проверяет, содержит ли ответ структурно корректную полезную нагрузку direct_result. Аргументы response — ответ для проверки (dict или JSON-строка) Возвращает True если ответ содержит валидный direct_result с непустым полем kind

escape_markdown(text: str, exclude_code_blocks: bool = True) -> str Экранирует специальные символы Markdown, с опциональным пропуском блоков кода. Аргументы text — исходный текст для экранирования exclude_code_blocks — исключать ли блоки кода из экранирования Возвращает Экранированный текст

get_image_size(image_path: str) -> tuple[int, int] Получает размеры изображения. Аргументы image_path — путь к файлу изображения Возвращает Кортеж (ширина, высота) изображения

resize_image_if_needed(image_path: str, max_dimension: int = 10000) -> tuple[io.BytesIO, str] Проверяет и при необходимости изменяет размеры изображения. Аргументы image_path — путь к изображению max_dimension — максимальный размер стороны изображения Возвращает Кортеж (BytesIO-объект с изображением, формат изображения)

handle_direct_result(config, update, response) Асинхронно обрабатывает прямой результат от плагина и направляет его в соответствующий метод отправки Telegram API. Аргументы config — конфигурация бота update — объект обновления Telegram (Update) response — ответ плагина (dict, строка JSON или другой тип) Возвращает список отправленных сообщений (Message) или пустой список при ошибке/отсутствии результата Исключения может возникать при ошибках сети, доступа к файлам или внутренних ошибках Telegram API; большинство перехватываются и логируются

cleanup_intermediate_files(response: any) Удаляет промежуточные файлы, созданные плагинами. Аргументы response — Ответ плагина в виде словаря или JSON-строки, содержащий путь к файлу для удаления.

encode_image(fileobj) Кодирует изображение в строку base64 с префиксом data URI. Аргументы fileobj — Объект файла (поддерживающий getvalue()) для кодирования. Возвращает Строку формата 'data:image/jpeg;base64,'.

decode_image(imgbase64) Декодирует изображение из строки base64 с data URI префиксом. Аргументы imgbase64 — Строка base64 с префиксом 'data:image/jpeg;base64,'. Возвращает Байтовую строку декодированного изображения.

markdown_to_html(text) Конвертирует Markdown-разметку в HTML с поддержкой основных элементов форматирования. Аргументы text — Исходный текст в формате Markdown. Возвращает HTML-строку с применённым форматированием (заголовки, списки, код, ссылки, жирный/курсив/зачёркнутый текст).

Аргументы config — Конфигурация бота. update — Объект обновления Telegram. response — Текст ответа для отправки. session_name — Базовое имя файла (по умолчанию 'response'). Возвращает Отправленное сообщение с документом.
