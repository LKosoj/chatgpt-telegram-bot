bot/chat_modes_registry

Источник: bot/chat_modes_registry.py

Модуль реализует реестр чат-режимов с "ленивой" загрузкой из YAML-файла и автоматической перезагрузкой при изменении файла. Поддерживает поиск режимов по ключу, по содержимому системного промпта (с точным и маркерным совпадением), получение списка режимов с приветственными сообщениями и валидацию требуемых инструментов.

Ключевые структуры данных ChatMode — Датакласс, представляющий чат-режим с ключом и ассоциированными данными.

class ChatMode Представляет чат-режим с идентификатором и конфигурационными данными. Поля key — Строковый идентификатор режима. data — Словарь с конфигурационными данными режима.

ChatModesRegistry.init(self, path: str) Конструктор реестра чат-режимов. Аргументы path — Путь к YAML-файлу с конфигурацией режимов.

ChatModesRegistry._load_if_needed(self) -> None Внутренний метод "ленивой" загрузки: проверяет существование файла, отслеживает mtime и перезагружает данные при изменении.

ChatModesRegistry.all_modes(self) -> Dict[str, Dict] Возвращает все загруженные чат-режимы. Возвращает Словарь, где ключ — идентификатор режима, значение — конфигурация режима.

ChatModesRegistry.get_mode_by_key(self, key: str) -> Optional[Dict] Получает режим по его строковому идентификатору. Аргументы key — Идентификатор чат-режима. Возвращает Конфигурацию режима или None, если режим не найден.

ChatModesRegistry.get_mode_by_system_prompt(self, system_content: str) -> Optional[Dict] Определяет режим по содержимому системного промпта. Сначала ищет точное совпадение по полю prompt_start, затем проверяет наличие всех маркеров из prompt_markers. Аргументы system_content — Текст системного промпта для сопоставления. Возвращает Конфигурацию подходящего режима или None.

ChatModesRegistry.get_all_modes_list(self) -> List[str] Формирует список строковых описаний режимов, содержащих приветственные сообщения. Возвращает Список строк формата "name: , welcome_message: ".

ChatModesRegistry.validate_tools(self, plugin_manager) -> None Проверяет, что все инструменты, указанные в конфигурациях режимов, доступны через plugin_manager. Логирует ошибки при обнаружении отсутствующих инструментов. Аргументы plugin_manager — Объект с методом has_plugin для проверки доступности плагинов.
