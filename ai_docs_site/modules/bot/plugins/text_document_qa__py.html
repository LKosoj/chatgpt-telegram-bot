bot/plugins/text_document_qa

Источник: bot/plugins/text_document_qa.py

Модуль реализует плагин документного Q&A на базе AnythingLLM с изоляцией данных по чатам. Каждый чат получает персональный workspace, куда загружаются документы; плагин предоставляет команды для просмотра списка документов, задания вопросов по их содержимому и удаления устаревших файлов. Включает фоновую задачу очистки документов старше 30 дней с предупреждением за сутки до удаления. Конфигурация осуществляется через переменные окружения с префиксом ANYTHINGLLM_.

Ключевые структуры данных AnythingLLMError — Исключение для ошибок, специфичных для взаимодействия с AnythingLLM API. processing_tasks — Dict[str, asyncio.Task] активных задач обработки документов workspace_map_cache — Dict[str, str] кэш соответствия chat_id → workspace slug _metadata_dir — Path директория хранения метаданных документов _http_transport — httpx.AsyncHTTPTransport конфигурация HTTP-транспорта

class AnythingLLMError Базовое исключение для ошибок операций с AnythingLLM.

class TextDocumentQAPlugin Плагин документного Q&A с бэкендом AnythingLLM, изолирующий документы по чатам.

Поля metadata_dir — str, путь к директории хранения метаданных документов. workspace_map_path — str, путь к JSON-файлу отображения чатов на workspace AnythingLLM. cleanup_task — asyncio.Task | None, фоновая задача периодической очистки устаревших документов. processing_tasks — dict[str, asyncio.Task], словарь активных задач обработки загружаемых документов. config — dict, конфигурация плагина (например, флаг enable_quoting). max_document_age — int, максимальный возраст документа в секундах (30 дней). warning_before_delete — int, интервал предупреждения перед удалением в секундах (1 день). _http_transport — httpx.AsyncBaseTransport | None, кастомный HTTP-транспорт для клиента.

Методы init(http_transport: httpx.AsyncBaseTransport | None = None) — Инициализация плагина с созданием необходимых директорий. initialize(openai=None, bot=None, storage_root: str | None = None) -> None — Запуск плагина с перенастройкой путей хранения и запуском фоновой очистки. get_source_name() -> str — Возвращает идентификатор источника "TextDocumentQA". get_commands() -> List[Dict] — Возвращает регистрацию CLI-команд плагина. get_spec() -> List[Dict] — Возвращает спецификацию функций для LLM-инструментов (upload_document, list_documents, ask_question, delete_document). execute(function_name: str, helper, **kwargs) -> Dict — Диспетчер выполнения функций плагина по имени операции.

execute(function_name: str, helper, **kwargs) -> Dict Центральный диспетчер обработки запросов к плагину.

Аргументы function_name — имя вызываемой функции (list_documents, upload_document, ask_question, delete_document). helper — вспомогательный объект выполнения. **kwargs — именованные аргументы включая chat_id, file_content, file_name, document_id, query, update.

Возвращает Dict с результатом операции или ошибкой; для успешных текстовых ответов использует _direct_text.

Исключения AnythingLLMError при сбоях взаимодействия с API; ошибки возвращаются в поле error ответа.
