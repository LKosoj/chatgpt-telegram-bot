bot/plugins/chief

Источник: bot/plugins/chief.py

ChiefPlugin — плагин для создания рецептов и кулинарных рекомендаций с использованием API Edamam и OpenAI. Предоставляет инструменты для поиска рецептов по ингредиентам и диетическим ограничениям, а также для планирования меню на заданное количество дней. Включает механизмы кэширования, повторных попыток при ошибках парсинга, автоматический перевод запросов с русского языка и форматирование результатов в Markdown с эмодзи.

Ключевые структуры данных query_schema — JSON Schema для валидации параметров поиска рецепта (ингредиенты, время приготовления, тип приёма пищи, диетические ограничения). menu_plan_schema — JSON Schema для валидации параметров плана питания (количество дней, предпочтения, исключаемые ингредиенты, типы приёмов пищи). _recipe_cache — внутренний словарь для кэширования результатов поиска рецептов с временем жизни 3 минуты. used_recipes — словарь для отслеживания уже использованных рецептов по типу приёма пищи; ключ — строка meal_type, значение — множество URL рецептов. meal_queries — словарь запросов по типам приёмов пищи для поиска рецептов (breakfast, lunch, dinner и др.). daily_menu — словарь сформированного меню на день; ключ — строка meal_type, значение — форматированная строка с рецептом и советами.

class ChiefPlugin Расширяет базовый класс Plugin для предоставления кулинарных функций через внешние API. Поля edamam_app_id — идентификатор приложения Edamam, загружается из переменной окружения EDAMAM_APP_ID. edamam_app_key — ключ приложения Edamam, загружается из переменной окружения EDAMAM_APP_KEY. edamam_user_id — идентификатор пользователя Edamam, инициализируется значением EDAMAM_APP_ID. query_schema — схема валидации для параметров поиска рецепта. menu_plan_schema — схема валидации для параметров плана питания. session — асинхронная HTTP-сессия (aiohttp.ClientSession), создаётся при необходимости. _recipe_cache — словарь для кэширования результатов API-запросов. _cache_timeout — время жизни кэша в секундах (по умолчанию 180). _api_timeout — таймаут API-запросов в секундах (по умолчанию 120). Методы init(self) — инициализирует плагин, загружает учётные данные Edamam из переменных окружения, проверяет их наличие, инициализирует схемы валидации и устанавливает параметры кэширования. get_source_name(self) -> str — возвращает строковый идентификатор источника "Chief". get_spec(self) -> List[Dict] — возвращает спецификацию доступных инструментов (get_recipe и plan_menu) в формате JSON Schema для регистрации в системе плагинов. _validate_credentials(self) — проверяет наличие всех обязательных учётных данных Edamam; выбрасывает ValueError при отсутствии любого из параметров. _init_schemas(self) — инициализирует JSON Schema для валидации параметров поиска рецепта и планирования меню. _translate_meal_type(self, meal_type: str) -> str — переводит тип приёма пищи с русского на английский; возвращает "other" при отсутствии соответствия. _translate_ingredients(self, ingredients: List[str], helper, user_id) -> Tuple[List[str], int] — асинхронно переводит список ингредиентов с русского на английский через LLM-хелпер; возвращает переведённые ингредиенты и количество использованных токенов. _parse_with_retry(self, user_query: str, helper, user_id: int, retries=3) -> Tuple[Dict, int] — парсит естественно-языковой запрос пользователя в структурированные параметры с несколькими попытками; использует LLM для извлечения ингредиентов, времени приготовления, типа приёма пищи и диетических ограничений; возвращает словарь параметров и количество токенов.

_ensure_session(self) Создаёт или восстанавливает HTTP-сессию aiohttp с настроенным таймаутом.

_get_cache_key(self, params: Dict) -> str Генерирует ключ кэша на основе параметров запроса. Аргументы params — словарь параметров запроса. Возвращает строковый ключ кэша.

_get_cached_recipes(self, cache_key: str) -> Optional[List] Получает рецепты из кэша если они есть и не устарели. Аргументы cache_key — ключ для поиска в кэше. Возвращает список кэшированных рецептов или None.

_cache_recipes(self, cache_key: str, recipes: List) Сохраняет рецепты в кэш с текущей временной меткой. Аргументы cache_key — ключ для сохранения. recipes — список рецептов для кэширования.

_search_recipes(self, params: Dict) -> List Поиск рецептов через API Edamam с кэшированием и обработкой ошибок. Аргументы params — валидированные параметры запроса (max_time, meal_type, dietary_restrictions, health_filters). Возвращает список найденных рецептов. Исключения TimeoutError — при превышении времени ожидания ответа API. aiohttp.ClientError — при сетевых ошибках.

_enhance_recipe(self, recipe, user_query, helper, user_id) Улучшает рецепт с помощью LLM: добавляет советы по приготовлению, замены ингредиентов, рекомендации по подаче. Аргументы recipe — словарь с данными рецепта (label, ingredientLines). user_query — исходный запрос пользователя. helper — объект для взаимодействия с LLM. user_id — идентификатор пользователя. Возвращает кортеж (response, tokens_used) — улучшенное описание и количество использованных токенов.

_format_output(self, recipes) Форматирует список рецептов в человекочитаемую строку для вывода пользователю. Аргументы recipes — список словарей с полями title, time, url, tips. Возвращает отформатированная строка с эмодзи-разметкой.

_generate_daily_menu(self, preferences: Dict, helper, user_id: int) -> Tuple[Dict, int] Генерирует меню на один день с учётом предпочтений пользователя. Аргументы preferences — словарь предпочтений (meal_types, dietary_preferences, _used_recipes). helper — объект для взаимодействия с LLM. user_id — идентификатор пользователя. Возвращает кортеж (daily_menu, total_tokens_used) — сгенерированное меню и суммарные затраты токенов. Исключения возможны исключения от _search_recipes и _enhance_recipe.

async def generate_daily_menu(self, preferences: Dict, meal_queries: Dict[str, str], helper, user_id: int, used_recipes: Dict[str, Set[str]]) -> Tuple[Dict[str, str], int] Генерирует меню на один день с учётом предпочтений и истории использованных рецептов. Аргументы preferences — структурированные предпочтения пользователя (dietary_preferences, excluded_ingredients, meal_types, health_filters, days). meal_queries — словарь запросов по типам приёмов пищи (breakfast, lunch, dinner и др.). helper — экземпляр LLM-ассистента для генерации советов. user_id — идентификатор пользователя. used_recipes — словарь для отслеживания уже использованных рецептов по типу приёма пищи. Возвращает кортеж (daily_menu, total_tokens_used), где daily_menu — словарь сформированного меню, total_tokens_used — общее количество использованных токенов LLM. Исключения TimeoutError — при превышении времени ожидания ответа от API рецептов. aiohttp.ClientError — при сетевых ошибках при обращении к API рецептов.

async def _parse_menu_preferences(self, preferences_str: str, helper, user_id: int, days: int = 7) -> Tuple[Dict, int] Парсит строку предпочтений пользователя в структурированный формат с помощью LLM. Аргументы preferences_str — строка с естественно-языковым описанием предпочтений пользователя. helper — экземпляр LLM-ассистента для анализа предпочтений. user_id — идентификатор пользователя. days — количество дней для планирования меню (по умолчанию 7). Возвращает кортеж (preferences, tokens_used), где preferences — валидированный словарь предпочтений, tokens_used — количество использованных токенов LLM. Исключения ValueError — при невозможности разобрать предпочтения или невалидном JSON из ответа LLM. json.JSONDecodeError — при ошибке парсинга JSON из ответа LLM. ValidationError — при несоответствии распарсенных предпочтений JSON Schema.

def _format_menu_plan(self, menu_plan: Dict[str, Dict[str, str]]) -> str Форматирует план меню на несколько дней в читаемый текст с эмодзи и русскими названиями приёмов пищи. Аргументы menu_plan — вложенный словарь плана меню; внешний ключ — номер дня, внутренний ключ — тип приёма пищи, значение — форматированная строка рецепта. Возвращает отформатированная строка с планом меню, готовая для отправки пользователю.

async def execute(self, function_name: str, helper, kwargs) -> Dict Центральный диспетчер выполнения команд сервиса рецептов. Аргументы function_name — имя вызываемой функции ("get_recipe" или "plan_menu"). helper — экземпляр LLM-ассистента для обработки запросов. kwargs — именованные аргументы, специфичные для вызываемой функции (query, user_id, days и др.). Возвращает словарь с результатом выполнения команды; для get_recipe содержит direct_result с форматированным markdown-текстом, для plan_menu — план меню на указанное количество дней.

self._parse_menu_preferences(preferences_str, helper, user_id, days) Парсит строку предпочтений пользователя в структурированный формат с помощью LLM. Аргументы preferences_str — строка с предпочтениями пользователя. helper — вспомогательный объект для взаимодействия с LLM. user_id — идентификатор пользователя для персонализации. days — количество дней планирования (передаётся в контекст парсинга). Возвращает кортеж (preferences, tokens_used), где preferences — структурированные предпочтения, tokens_used — количество потраченных токенов.

self._generate_daily_menu(preferences, helper, user_id) Генерирует меню на один день на основе распарсенных предпочтений. Аргументы preferences — структурированные предпочтения пользователя. helper — вспомогательный объект для взаимодействия с LLM. user_id — идентификатор пользователя для персонализации. Возвращает кортеж (daily_menu, tokens_used), где daily_menu — сгенерированное меню на день, tokens_used — количество потраченных токенов.

self._format_menu_plan(menu_plan) Форматирует словарь меню по дням в итоговую строку Markdown. Аргументы menu_plan — словарь {day_number: daily_menu}. Возвращает отформатированная строка плана питания в Markdown.

self.t(key, kwargs) Локализует сообщение об ошибке по ключу. Аргументы key — ключ строки локализации. kwargs — параметры подстановки в шаблон. Возвращает локализованная строка с подставленными значениями.
