bot/plugins/skills

Источник: bot/plugins/skills.py

Модуль реализует плагин SkillsPlugin, который интегрирует локальные Codex-стилизованные навыки (skills) в цикл function-calling бота. Плагин предоставляет инструменты для обнаружения, установки, активации и выполнения навыков с поддержкой скриптов, аудита и управления состоянием через файловую систему и CLI. Конфигурация осуществляется через переменные окружения и файлы состояния в хранилище бота.

Ключевые структуры данных SkillsPlugin — основной плагин, управляющий жизненным циклом навыков: сканирование директорий, загрузка метаданных, выполнение скриптов, аудит операций и отражения (reflections) для оптимизации повторных вызовов. Plugin — базовый класс плагина, от которого наследуется SkillsPlugin.

class SkillsPlugin Плагин для экспозиции локальных Codex-стилевых навыков как инструментов function-calling.

Поля skills_dir — директория хранения навыков (из SKILLS_DIR или storage_root/skills). state_file — путь к JSON-файлу состояния активных навыков. reflections_file — путь к JSON-файлу кэшированных отражений навыков. workdir_root — корневая рабочая директория для выполнения скриптов навыков. audit_file — путь к JSONL-файлу аудита операций. available_skills — словарь обнаруженных навыков с метаданными. active_skills — словарь активных навыков по scope (chat/user). skill_reflections — словарь отражений навыков по scope. allow_scripts — флаг разрешения выполнения скриптов (SKILLS_ALLOW_SCRIPTS). allow_installs — флаг разрешения установки навыков (SKILLS_ALLOW_INSTALLS). script_admin_ids — множество ID администраторов для скриптов. script_admin_all — флаг разрешения скриптов для всех пользователей. install_admin_ids — множество ID администраторов для установки. install_admin_all — флаг разрешения установки для всех пользователей. script_timeout — таймаут выполнения скриптов в секундах. install_timeout — таймаут установки навыков в секундах. output_max_chars — максимальное количество символов в выводе скрипта. interim_after_seconds — интервал отправки промежуточных сообщений. _state_lock — RLock для синхронизации доступа к состоянию. _reflection_lock — RLock для синхронизации отражений. _audit_lock — Lock для синхронизации аудита. _scan_cache — кэш результатов сканирования навыков.

Методы init() — инициализация плагина с настройками по умолчанию. initialize(openai=None, bot=None, storage_root: str | None = None) -> None — инициализация путей, загрузка конфигурации из окружения, сканирование навыков и загрузка состояния. get_source_name() -> str — возвращает идентификатор источника "Skills". get_spec() -> List[Dict] — возвращает спецификации инструментов function-calling для навыков.

Модуль реализует инструменты для управления навыками агента: перечисление доступных и активных навыков, установка из внешних источников, активация с контекстом, обновление прогресса по шагам, выполнение разрешённых скриптов, запись рефлексий после ошибок и деактивация. Все операции проверяют права пользователя и состояние готовности компонента.

Ключевые структуры данных disabled_skills — множество строковых идентификаторов навыков, отключённых пользователем через настройки.

Возвращает Словарь со списком навыков или информацию об ошибке.

Возвращает Результат установки или запрос подтверждения.

Аргументы skill_name — идентификатор директории навыка или имя из метаданных. initial_context — начальный контекст выполнения (строка/объект, опционально). Аргументы skill_name — идентификатор директории навыка или имя из метаданных. disabled_skills — множество отключённых навыков для проверки доступа. Модуль реализует инфраструктуру для обнаружения, сканирования и управления навыками, представленными директориями с Markdown-описаниями и исполняемыми скриптами. Поддерживает кэширование сканирования по сигнатуре времени модификации, парсинг frontmatter в форматах YAML и плоском ключ-значение, явное объявление точек входа через метаданные и эвристическое определение по расширению/исполняемости файлов. Обеспечивает валидацию безопасности путей и фильтрацию отключённых навыков.

Ключевые структуры данных _scan_cache — кортеж (signature, skills), хранящий кэшированный результат сканирования и его сигнатуру для инвалидации. available_skills — словарь обнаруженных навыков с полными метаданными, содержимым и скриптами.

_check_disabled_skill(skill_name: str, disabled_skills: set[str]) -> Dict[str, Any] | None Проверяет, отключён ли указанный навык в пользовательских настройках. Аргументы skill_name — имя или идентификатор навыка для проверки. disabled_skills — множество идентификаторов отключённых навыков. Возвращает Словарь с ошибкой, если навык отключён; None, если навык доступен.

_scan_skills() -> Dict[str, Dict[str, Any]] Выполняет полное сканирование директории навыков с построением кэша. Аргументы нет. Возвращает Словарь идентификаторов навыков в их полные описания.

_scan_signature() -> tuple Формирует сигнатуру изменений для инвалидации кэша сканирования. Аргументы нет. Возвращает Кортеж записей (имя_навыка, mtime_маркдауна, точки_входа_скриптов) или пустой кортеж при ошибке.

_parse_skill_markdown(content: str) -> tuple[Dict[str, Any], str] Разбирает содержимое SKILL.md на метаданные frontmatter и тело документа. Аргументы content — сырое содержимое Markdown-файла. Возвращает Кортеж (метаданные, тело_документа); метаданные пустые при отсутствии frontmatter.

_parse_flat_frontmatter(frontmatter: str) -> Dict[str, Any] Парсит плоский формат frontmatter без YAML-парсера (ключ: значение с продолжением строк). Аргументы frontmatter — текстовое содержимое секции frontmatter. Возвращает Словарь извлечённых метаданных.

_parse_flat_scalar(value: str) -> Any Преобразует строковое значение из плоского frontmatter в типизированное. Аргументы value — строковое представление значения. Возвращает Распакованная строка без кавычек, булево значение или исходную строку.

_list_skill_scripts(skill_path: Path, metadata: Dict[str, Any] | None = None) -> List[str] Формирует список точек входа скриптов навыка с приоритетом явных объявлений. Аргументы skill_path — путь к директории навыка. metadata — опциональные метаданные из frontmatter для разрешения entrypoints. Возвращает Отсортированный список относительных POSIX-путей скриптов.

_resolve_explicit_entrypoints(metadata: Dict[str, Any], scripts_dir: Path) -> List[str] | None Разрешает явно объявленные точки входа из метаданных с валидацией безопасности. Аргументы metadata — словарь метаданных навыка. scripts_dir — путь к директории scripts навыка. Возвращает Список валидных точек входа, пустой список при null, None для fallback к эвристике. Исключения Логирует предупреждения при невалидных entrypoints; не выбрасывает исключений.

_is_script_entrypoint(path: Path, scripts_dir: Path) -> bool Определяет, является ли файл кандидатом на точку входа по эвристикам. Аргументы path — путь к проверяемому файлу. scripts_dir — базовая директория scripts для вычисления относительного пути. Возвращает True, если файл проходит фильтры расширения/исполняемости и не исключён.

_list_skills(*, refresh: bool = False, disabled_skills: set[str] | None = None) -> Dict[str, Any] Формирует ответ со списком доступных навыков с учётом кэша и фильтрации. Аргументы refresh — принудительное обновление кэша сканирования. disabled_skills — множество идентификаторов для исключения из результата. Возвращает Словарь с флагом success и списком навыков.

Модуль реализует асинхронный менеджер навыков агента с возможностью поиска, установки и синхронизации пакетов через внешний CLI skills. Включает механизмы контроля доступа, валидации параметров, обработки ошибок и аудита операций.

Ключевые структуры данных Dict[str, Any] — стандартный формат возвращаемого результата операций, содержащий поля success, error, stdout, stderr и специфичные для операции данные.

async def _find_installable_skills(self, query: str) -> Dict[str, Any] Поиск доступных для установки навыков через CLI по заданному запросу. Аргументы query — строка поискового запроса; пустая или содержащая только пробелы вызывает ошибку. Возвращает словарь с результатами поиска: success, query, results, stdout, stderr, returncode.

async def _install_skill(self, package: str, , skill_name: str | None, confirmed: bool, *kwargs) -> Dict[str, Any] Установка навыка из указанного пакета с валидацией прав и синхронизацией в рабочую директорию. Аргументы package — идентификатор пакета навыка для установки. skill_name — желаемое имя навыка в системе; если None, выводится из package. confirmed — флаг явного подтверждения пользователя для установки. kwargs — дополнительные параметры, включая user_id для проверки прав. Возвращает словарь с результатом установки: success, skill, package, path, sync, stdout, stderr, available. Исключения возвращает ошибку в поле error результата при: запрете установок, отсутствии прав администратора, неподтверждённой операции, невалидном package, отсутствии skill_name, занятом имени, ошибке CLI или синхронизации.

def _sync_installed_skill(self, source_path: Path, target_path: Path) -> tuple[str, str | None] Синхронизация установленного навыка из исходного пути в целевую директорию навыков. Аргументы source_path — путь к установленному навыку в системе. target_path — путь назначения в директории навыков агента. Возвращает кортеж (mode, error): mode описывает способ синхронизации ("already_in_skills_dir", "copied_to_skills_dir" или пустая строка при ошибке), error содержит сообщение об ошибке или None. Исключения OSError при неудаче копирования файловой иерархии.

async def _run_skills_cli(self, args: List[str], *, timeout: int) -> Dict[str, Any] Запуск внешнего CLI skills через npx с заданными аргументами и таймаутом. Аргументы args — список аргументов командной строки для передачи в skills. timeout — максимальное время ожидания выполнения команды в секундах. Возвращает словарь с результатом выполнения: success, returncode, stdout, stderr. Исключения asyncio.TimeoutError при превышении таймаута — процесс принудительно завершается. Exception при ошибке запуска процесса.

Внутренние вспомогательные методы для управления навыками (skills). Реализуют парсинг вывода CLI, поиск путей установленных навыков, валидацию имён директорий, очистку ANSI-escape последовательностей, управление состоянием активных навыков и генерацию вызовов скриптовых инструментов.

Внутренние методы менеджера навыков для управления состоянием, прогрессом и скриптовым выполнением.

Ключевые структуры данных active_skills — потокобезопасное хранилище состояний активных навыков, индексированное по скоупу (chat_id + user_id). scope_state — состояние навыков в рамках одного скоупа: {skill_id: state_dict}. state — словарь состояния навыка с полями current_step, activated_at, updated_at, context.

_get_skill_status(skill_name: str = "", disabled_skills: set[str] | None = None, **kwargs) -> Dict[str, Any] Возвращает статус активных навыков в скоупе или конкретного навыка. Аргументы skill_name — имя навыка для проверки; пустая строка возвращает все активные навыки. disabled_skills — множество идентификаторов скрываемых навыков. kwargs — должен содержать chat_id и/или user_id для вычисления скоупа. Возвращает словарь с success, scope, active_skills/visible_scope_state, active_script_tool_calls (при skill_name="") или success, scope, skill, state, script_tool_calls (при указанном навыке). Исключения не бросает исключений; ошибки возвращаются в поле error результата.

_update_skill_progress(skill_name: str, step: int, context_update: Any = None, **kwargs) -> Dict[str, Any] Обновляет текущий шаг и контекст активного навыка. Аргументы skill_name — имя обновляемого навыка. step — новое значение текущего шага. context_update — обновляемые данные контекста (декодируются через _decode_context). kwargs — должен содержать chat_id и/или user_id для вычисления скоупа. Возвращает словарь с success, scope, skill, state при успехе; success=False и error при неудаче. Исключения не бросает исключений; ошибки возвращаются в поле error результата.

_deactivate_skill(skill_name: str, **kwargs) -> Dict[str, Any] Деактивирует навык в скоупе и очищает пустые скоупы. Аргументы skill_name — имя деактивируемого навыка. kwargs — должен содержать chat_id и/или user_id для вычисления скоупа. Возвращает словарь с success, scope, skill, deactivated=True при успехе; success=False и error при неудаче. Исключения не бросает исключений; ошибки возвращаются в поле error результата.

_run_skill_script(skill_name: str, script_name: str, args_json: Any = None, **kwargs) -> Dict[str, Any] Асинхронно выполняет скрипт навыка с контролем безопасности и таймаутом. Аргументы skill_name — имя навыка, содержащего скрипт. script_name — имя исполняемого файла скрипта. args_json — аргументы скрипта в JSON-представлении. kwargs — должен содержать chat_id и/или user_id для вычисления скоупа. Возвращает словарь с результатом выполнения: success, stdout, stderr, returncode, duration_ms при успехе; success=False и error при неудаче. Исключения asyncio.TimeoutError — перехватывается внутри, возвращается error="timeout". OSError — при ошибках доступа к файлу скрипта. ValueError — при невалидных аргументах args_json.

Менеджер навыков с поддержкой рефлексии и аудита. Обеспечивает сканирование каталогов навыков, запуск скриптов в изолированных рабочих директориях, отслеживание состояния активных навыков по scope, а также накопление предложений по улучшению с автоматической записью в SKILL.md при превышении порога повторений.

Ключевые структуры данных REFLECTION_REPEAT_THRESHOLD — пороговое значение количества повторений рефлексии, при достижении которого предложение автоматически применяется к SKILL.md.

_cleanup_after_delivery(cleanup: Dict[str, Any]) -> bool Удаляет навык из активного состояния указанного scope после завершения доставки результата. Аргументы cleanup — словарь с ключами scope и skill_id, идентифицирующими удаляемый экземпляр навыка. Возвращает True если состояние было найдено и удалено, иначе False.

Аргументы script_path — путь к исполняемому файлу скрипта. Аргументы skill_name — имя или идентификатор навыка для поиска. Возвращает Найденный skill_id или None при отсутствии совпадений.

_decode_context(value: Any) -> Dict[str, Any] Нормализует входное значение контекста в словарь, поддерживая None, dict, JSON-строку и произвольный текст. Аргументы value — исходное значение контекста произвольного типа. Возвращает Словарь с декодированным контекстом; при нераспознанном тексте возвращает {"task": text}.

_parse_script_args(args_json: Any) -> List[str] Парсит аргументы скрипта из JSON-строки, списка или словаря в плоский список строковых аргументов. _record_skill_reflection(skill_name: str, proposal: str, , failure_mode: Any = None, evidence: Any = None, kwargs) -> Dict[str, Any] Регистрирует предложение по улучшению навыка, накапливает счётчик повторений и при превышении порога автоматически применяет изменение к SKILL.md. Аргументы skill_name — имя или идентификатор навыка для рефлексии. proposal — текст предложения по улучшению или уточнению. failure_mode — опциональное описание режима отказа, вызвавшего рефлексию. evidence — опциональные доказательства/пример для сохранения в истории. *kwargs — дополнительные параметры включая chat_id и user_id для формирования scope. Аргументы skill_id — идентификатор навыка для модификации. proposal_text — текст предложения для добавления в документацию. Возвращает Словарь с полями applied, already_present или error при неудаче чтения/записи. Исключения Не выбрасывает исключений; ошибки ОС возвращаются в поле error результата.

def _spawn_interim_notice(self, chat_id: Any, skill_id: str, script_name: str) -> "asyncio.Task | None" Запускает фоновую задачу уведомлений о длительном выполнении скрипта. Аргументы chat_id — идентификатор чата (приводится к int). skill_id — идентификатор навыка. script_name — имя выполняемого скрипта. Возвращает asyncio.Task или None если уведомления отключены или бот недоступен.

Вспомогательные утилиты для управления жизненным циклом скриптов навыков: watchdog-уведомления, аудит, обрезка вывода и проверка путей.

_watchdog() Внутренняя корутина, реализующая фоновый мониторинг выполнения скрипта навыка. Аргументы отсутствуют (замыкание на внешние переменные: bot, chat_id_int, script_name, skill_id, typing_refresh_seconds, first_delay, repeat_delay). Возвращает None (бесконечный цикл до отмены задачи). Исключения asyncio.CancelledError — пробрасывается выше для корректной отмены задачи; прочие исключения логируются и подавляются.

create_task(_watchdog()) Создаёт и возвращает фоновую задачу watchdog для активного event loop. Аргументы отсутствуют. Возвращает asyncio.Task | None — задача watchdog или None, если нет активного event loop. Исключения отсутствуют (RuntimeError при отсутствии активного loop перехватывается и возвращается None).

_audit(action, fields) Записывает аудит-событие в файл в формате JSON Lines. Аргументы action — строка, идентификатор действия. fields — произвольные именованные аргументы для включения в запись (значения None пропускаются). Возвращает None. Исключения отсутствуют (ошибки записи логируются и подавляются).

_truncate(value) Обрезает строку до максимально допустимой длины с добавлением суффикса о количестве пропущенных символов. Аргументы value — исходная строка для обрезки. Возвращает str — исходная строка, если её длина не превышает output_max_chars; иначе обрезанная строка с суффиксом "\n...[truncated {omitted} chars]". Исключения отсутствуют.

_is_relative_to(path, parent) Проверяет, находится ли путь внутри указанного родительского каталога. Аргументы path — Path, проверяемый путь. parent — Path, родительский каталог для проверки. Возвращает bool — True, если path является подпутем parent; False в противном случае. Исключения отсутствуют (ValueError от path.relative_to перехватывается и преобразуется в False).
