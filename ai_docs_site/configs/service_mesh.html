Service Mesh / Ingress

LLM Gateway Client

Асинхронный HTTP-клиент для взаимодействия с LLM Gateway — прокси-сервисом унифицированного доступа к LLM- и мультимодальным возможностям в рамках service mesh.

Конфигурация клиента

from bot.llm_gateway_client import LLMGatewayClient, LLMGatewayError

client = LLMGatewayClient(
    base_url="https://gateway.mesh.svc/v1",  # Обязательный параметр
    api_key="sk-...",                         # Bearer-токен
    timeout=120.0                             # Глобальный таймаут, сек
)
Параметр Требование Описание
base_url Обязательный Корневой URL шлюза; LLMGatewayError при отсутствии
api_key Обязательный Вставляется в заголовок Authorization: Bearer <api_key>
timeout Опциональный (по умолчанию 120.0) Переопределяется для конкретных методов

Переопределение таймаутов по операциям

Метод Таймаут Причина
web_deep_research 1800.0 Многошаговое исследование с рекурсивным анализом
web_research, image_edit 300.0 Генерация контента и обработка мультимодальных данных
web_search 90.0 Простой поисковый запрос
Остальные методы 120.0 Значение по умолчанию

Идентификационный заголовок: X-Title: tgBot — передаётся во всех запросах для трассировки и rate limiting.

Низкоуровневые методы

# JSON-запросы с валидацией ответа
await client.get_json(endpoint: str, params: dict | None = None)
await client.post_json(endpoint: str, json: dict)

# Multipart-загрузка для бинарных данных
await client.post_multipart(
    endpoint: str,
    data: dict,           # текстовые поля формы
    files: dict           # файлы: {field: (filename, bytes, mime_type)}
)

Все методы: - Проверяют HTTP-статус; при ошибке поднимают LLMGatewayError с полным текстом ответа - Включают заголовки авторизации и X-Title

Мультимодальные операции

# Веб-поиск
results = await client.web_search(
    query="...",
    max_results=10,           # ограничение результатов
    language="ru",            # языковой фильтр
    include_images=True       # включить изображения в выдачу
)

# Чтение страницы в markdown
page = await client.web_read(url="https://...", format="markdown")

# Исследование: выбор режима через классификацию
research = await client.web_research(query="...")
deep_research = await client.web_deep_research(
    query="...",
    max_words=5000,           # объём финального отчёта
    breadth=4,                # число параллельных направлений
    depth=2,                  # глубина рекурсии
    concurrency=3             # одновременных подзапросов
)

Редактирование изображений

# Из base64-строки
result = await client.image_edit(
    image_base64="data:image/png;base64,...",
    prompt="Добавить фон..."
)

# Из файла — multipart-загрузка
result = await client.image_edit_file(
    file_path="/tmp/image.png",
    prompt="Изменить стиль..."
)

Модель определяется константой LLMGATEWAY_IMAGE_EDIT_MODEL из bot.model_constants.

Утилита извлечения результата

from bot.llm_gateway_client import extract_image_result

# Вариант 1: URL в ответе — возвращает строку
url = extract_image_result({"url": "https://cdn.mesh/result.png"})

# Вариант 2: Base64 — сохраняет во временный файл, возвращает путь
path = extract_image_result({"b64_json": "iVBORw0KGgo..."})
# /tmp/llm_img_7f3a9e2d.png

Обработка ошибок

class LLMGatewayError(Exception):
    """Конфигурационные ошибки (отсутствует base_url) и HTTP-ошибки"""

    # Примеры возникновения:
    # - LLMGatewayClient(base_url=None)  → "base_url is required"
    # - HTTP 502 с телом 700+ символов    → полный текст включён в сообщение

Маршрутизация трафика (Path)

Файл конфигурации path определяет правила сопоставления входящих запросов с целевыми сервисами в service mesh.

Структура правила

- match:
    prefix: "/v1/llm/"          # или exact: "/health", regex: "^/api/.*"
    method: POST                # фильтр по HTTP-методу
    headers:
      X-Title: "tgBot"          # условие по метаданным
      Authorization: "Bearer .*" # regex в значениях заголовков
  route:
    destination: llm-gateway    # идентификатор сервиса в mesh
    weight: 100                 # 100% трафика (canary: 90/10)
  timeout: 120s
  retry:
    attempts: 3
    perTryTimeout: 30s
    retryOn: gateway-error,connect-failure
  # или redirect/rewrite вместо route

Приоритет и распространение

Аспект Поведение
Приоритет Первое совпавшее правило (порядок определения критичен)
Обновление Через control plane без перезапуска sidecar-прокси
Зеркалирование mirror — копия трафика на тестовый сервис без влияния на клиента

Типы сопоставления путей

Тип Применение Пример
prefix API-версии, группы эндпоинтов /v1/llm/, /v1/audio/
exact Health checks, статические пути /health, /ready
regex Сложные паттерны ^/v\d+/images/.*

Распределение трафика (canary / blue-green)

route:
  - destination: llm-gateway-v2
    weight: 10        # canary — 10% новой версии
  - destination: llm-gateway-v1
    weight: 90

Хаос-тестирование (fault injection)

fault:
  delay:
    percentage: 5.0
    fixedDelay: 10s   # 5% запросов с задержкой 10 секунд
  abort:
    percentage: 0.1
    httpStatus: 503   # 0.1% запросов — ошибка 503

Валидация корректности работы WebResearchPlugin и OpenAIHelper.classify_reply_intent при интеграции с LLM Gateway.

assert call.max_tokens == 1000

Сценарии веб-исследований
