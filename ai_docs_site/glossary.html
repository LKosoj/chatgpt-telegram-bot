Глоссарий

Системные компоненты

AGENTS.md

Файл проектных инструкций для AI-агентов, работающих с кодовой базой. Определяет рабочие правила: формулировка предположений перед правками, использование rg для поиска, проверка git status --short, цитирование file:line. Содержит структуру проекта, процесс навигации через .cli-proxy/.codebase_map/, правила работы с плагинами, инструментами, режимами чата, обработчиками Telegram и SQLite. Приоритет кода выше README при расхождениях; исторические планы выносятся в датированные документы.

Codebase Mapper Graph

Механизм навигации и поддержания актуальности документации через графовую структуру инкрементально загружаемых узлов. Точка входа: /.cli-proxy/.codebase_map/INDEX.md; узлы: /.cli-proxy/.codebase_map/nodes/*.md; абсолютный путь развёртывания: /srv/git_projects/chatgpt-telegram-bot/.cli-proxy/.codebase_map. При модификации кода обновляется поле Last reviewed; при неудаче — запуск ремонта через update-node или repair. Содержит машиночитаемый маркер <!-- CODEBASE_MAPPER_GRAPH:END -->.

LLMGateway

Трёхуровневый маршрутизатор моделей (high / light_model / big_context) с настраиваемыми LLMGATEWAY_*_MODEL и базовым URL. Единая точка доступа к LLM- и мультимодальным возможностям (веб-поиск, чтение страниц, исследования, редактирование изображений, аудио) в рамках service mesh.

LLMGatewayClient

Асинхронный HTTP-клиент на базе httpx.AsyncClient. Критические настройки: base_url — обязателен (LLMGatewayError при отсутствии), api_key в заголовке Authorization: Bearer, глобальный timeout 120.0 с с переопределениями (web_deep_research 1800.0, web_research/image_edit 300.0, web_search 90.0), идентификационный заголовок X-Title: tgBot.

Архитектура runtime

Точка входа

bot/__main__.py — запуск бота с валидацией переменных окружения и инициализацией компонентов.

Последовательность инициализации

PluginManager → Database → OpenAIHelper → ChatGPTTelegramBot.

Telegram-polling

Реализован в bot/telegram_bot.py с базовым URL http://localhost:8081/bot при TELEGRAM_LOCAL_MODE=true.

Ключевые модули

ChatGPTTelegramBot

Обработчик Telegram-обновлений с inline-режимом и сессионным управлением. Обрабатывает команды, inline-запросы, callback-запросы и медиасообщения с буферизацией, потоковой передачей ответов, блокировками для потокобезопасности.

OpenAIHelper

Клиент для чата, vision, генерации изображений, TTS, транскрипции. Содержит механизмы повторных попыток при превышении лимитов API, суммаризации длинных контекстов, адаптацию форматов истории под различных провайдеров (OpenAI, Anthropic, Google, Mistral и др.).

OpenAIToolHandler

Полный жизненный цикл обработки вызовов функций (tool calls): парсинг и валидация ответов модели, маршрутизация к плагинам, немедленная доставка пользователю, отложенная передача в контекст модели с компактизацией, слияние множественных результатов, принудительная доставка через agent_tools.deliver_to_user в режиме skills_agent.

PluginManager

Система плагинов с namespace-инструментами, JSON-схемами и ~43 встроенными модулями. Механизмы разрешения имён функций, валидации разрешённых плагинов, нормализации спецификаций, управления жизненным циклом экземпляров с поддержкой кеширования. Интеграция с OpenAI API через формирование спецификаций функций, включая сценарии делегирования субагентам с явными списками блокировки.

Database

Singleton Database с thread-local соединениями SQLite, WAL, busy_timeout. JSON-структура conversation_context.context как {"messages": [...]}. Подсчёт message_count по сообщениям роли user. Ограничение max_sessions с удалением старейшей сессии. Длительные вызовы OpenAI вынесены за пределы транзакций.

ChatModesRegistry

Загрузчик и валидатор YAML-конфигурации bot/chat_modes.yml. Индексирует режимы по id; ChatMode содержит system_prompt, model_profile, plugins_allowlist/plugins_denylist, budget, feature-флаги, agent_limits, tts_voice, response_format. Валидация: уникальность id, разрешённые model_profile, корректность ссылок на плагины, валидность regex; при STRICT_MODE=true ошибки блокируют старт.

Подсистемы

Плагины и инструменты

Базовый класс: Plugin в bot/plugins/plugin.py.

Идентификация: по plugin_id, неймспейсинг функций через function_prefix.

Загрузка: из bot/plugins/*.py с учётом allow-list PLUGINS.

Валидация дубликатов: строгая/мягкая через PLUGIN_STRICT_VALIDATION.

Выполнение: пакетное через asyncio.gather.

Инъекция контекста: chat_id/user_id в вызовы плагинов.

Прямой ответ плагина: обход повторного вызова модели.

Форматы спецификаций: различие для Google и прочих моделей.

Режимы чата

Декларация в bot/chat_modes.yml. Валидация tool-ссылок при инициализации. Ограничение доступных плагинов через поле tools режима (по умолчанию ['All']). Требование использования имён модулей плагинов.

Обработчики Telegram

Нормализация команд через PluginManager.get_plugin_commands(). Запрет пробелов в именах команд. Обработка фильтров сообщений без eval.

MCP (Model Context Protocol)

Подключение удалённых MCP-серверов как динамических инструментов модели. Транспорты: HTTP (REST API) и stdio (дочерние процессы через stdin/stdout). Конфигурация: bot/config/mcp_servers.json. Регистрация требует server_name, user_id; для HTTP — base_url, опционально api_key; для stdio — transport: "stdio", command, args, опционально env. Управление: регистрация и удаление — только администраторы; просмотр списка (/mcp_servers) — все пользователи.

Hindsight

Опциональная долгосрочная память с автовызовом/автосохранением при удалении сессии. Идентификация банка: telegram-{user_id}. Типы памяти: world, experience, profile, session preference. Метаданные с mode: "session_close".

Agent runtime (agent_tools)

Подагенты (до 5 параллельно, 10–50 раундов), планировщик задач (TASKS_TTL_SECONDS=2 дня), взаимодействие с пользователем.

Skills runtime (skills)

Локальные Codex-стиль папки SKILL.md с выполнением скриптов. Структура: <skill_name>/SKILL.md (YAML frontmatter: name, description, allow_scripts, entrypoints + markdown body) и scripts/ с исполняемыми файлами.

Конфигурация и переменные окружения

Обязательные переменные

  • TELEGRAM_BOT_TOKEN
  • OPENAI_API_KEY
  • LLMGATEWAY_BASE_URL (или OPENAI_API_BASE)

Ключевые секции конфигурации

Секция Критические параметры
Telegram Core TELEGRAM_LOCAL_MODE (true → localhost:8081)
LLM/Маршрутизация LLMGATEWAY_BASE_URL, *_MODEL для трёх уровней, OPENAI_BASE_URL
Мультимодальность ENABLE_* флаги; VISION_MODEL, TTS_MODEL/TTS_VOICE, TRANSCRIPTION_MODEL, IMAGE_GENERATION_MODEL/IMAGE_EDIT_MODEL с качеством/размером/стилем
Сессии MAX_HISTORY_SIZE, MAX_CONVERSATION_AGE_MINUTES, MAX_SESSIONS (5), AUTO_CHAT_MODES
Бюджеты BUDGET_PERIOD (monthly/daily/weekly/total), USER_BUDGETS (* = безлимит), GUEST_BUDGET, ценообразование по токенам/изображениям/TTS/vision/транскрипции
Плагины PLUGINS (CSV allow-list), PLUGINS_DIR, PLUGIN_STRICT_VALIDATION, PLUGIN_STORAGE_ROOT (data/), PLUGIN_MENU_PAGE_SIZE (8), SHOW_PLUGINS_USED
Agent Tools AGENT_ASK_USER_TIMEOUT_SECONDS (1800); жёстко: MAX_SUBAGENTS (5), MIN/MAX_SUBAGENT_TOOL_ROUNDS (10/50), DELIVERY_DEDUP_WINDOW_SECONDS (60), DELIVERY_MAX_ARTIFACT_BYTES (49 MiB), блокировка 4 инструментов самозащиты для подагентов

MCP MCP_SERVERS (JSON-конфиг), DEFAULT_MCP_SERVERS, MCP_SERVERS_ALLOWED_USERS, MCP_REQUEST_TIMEOUT
Hindsight HINDSIGHT_ENABLED, HINDSIGHT_RECALL_ON_START, HINDSIGHT_SAVE_ON_DELETE, HINDSIGHT_API_URL/HINDSIGHT_BASE_URL, HINDSIGHT_API_TOKEN, HINDSIGHT_AUTO_RECALL/AUTO_SAVE, HINDSIGHT_MEMORY_TYPES, HINDSIGHT_MAX_AUTO_SAVE_ITEMS

Docker и инфраструктура

docker-compose.yml

Сервис chatgpt-telegram-bot собирается из Dockerfile в текущей директории (контекст .). Volume-монтирование директории проекта в /app для горячей перезагрузки кода при разработке. Политика перезапуска: unless-stopped (автоматически при сбоях или перезагрузке системы, кроме ручной остановки).

Dockerfile

Базовый образ: python:3.9-alpine. Системный пакет: ffmpeg. Точка входа: python -m bot. Переменные окружения: PYTHONFAULTHANDLER=1, PYTHONUNBUFFERED=1, PYTHONDONTWRITEBYTECODE=1, PIP_DISABLE_PIP_VERSION_CHECK=on. Флаги --no-cache для apk и --no-cache-dir для pip — минимизация размера слоёв.

Конфигурация pytest
