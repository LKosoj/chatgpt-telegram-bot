Обзор проекта

Файл docker-compose.yml определяет конфигурацию сервиса для запуска Telegram-бота с интеграцией ChatGPT в контейнере Docker.

Сервис chatgpt-telegram-bot собирается из Dockerfile в текущей директории (контекст .). Используется volume-монтирование директории проекта в контейнер по пути /app для обеспечения горячей перезагрузки кода при разработке.

Контейнер автоматически перезапускается при сбоях или перезагрузке системы, за исключением случаев, когда остановлен вручную (restart: unless-stopped).

Ключевые параметры: - build.context: указывает на текущую директорию как источник сборки. - build.dockerfile: используемый файл сборки — Dockerfile. - volumes: проброс директории с хоста в контейнер для разработки. - restart: политика перезапуска контейнера.

Файл AGENTS.md — проектные инструкции для AI-агентов, работающих с кодовой базой Python Telegram-бота с интеграцией OpenAI.

Назначение. Определяет рабочие правила агента (проверка состояния репозитория, хирургичность изменений, верификация утверждений через код), структуру проекта, процесс навигации по кодовой базе через .cli-proxy/.codebase_map/, а также специфические правила работы с плагинами, инструментами (tool calls), режимами чата, обработчиками Telegram и базой данных SQLite.

Ключевые сущности и правила.

  • Рабочий процесс: перед нетривиальными правками — формулировка предположений, критериев успеха и плана верификации; использование rg для поиска; проверка git status --short; цитирование file:line при объяснении поведения.
  • Архитектура runtime: точка входа bot/__main__.py, обязательные переменные окружения TELEGRAM_BOT_TOKEN и OPENAI_API_KEY, последовательность инициализации PluginManager → Database → OpenAIHelper → ChatGPTTelegramBot, Telegram-polling в bot/telegram_bot.py с базовым URL http://localhost:8081/bot.
  • Поток обработки запросов: telegram_bot.py (обработка обновлений), openai_helper.py (API-вызовы), openai_tool_handler.py (выполнение tool calls), plugin_manager.py (плагины и валидация), database.py (SQLite), chat_modes_registry.py + chat_modes.yml (режимы чата).
  • Плагины и инструменты: базовый класс Plugin в bot/plugins/plugin.py, идентификация по plugin_id, неймспейсинг функций через function_prefix, загрузка из bot/plugins/*.py с учётом allow-list PLUGINS, строгая/мягкая валидация дубликатов через PLUGIN_STRICT_VALIDATION, пакетное выполнение через asyncio.gather, инъекция chat_id/user_id, прямой ответ плагина для обхода повторного вызова модели, различие форматов спецификаций для Google и прочих моделей.
  • Режимы чата: декларация в bot/chat_modes.yml, валидация tool-ссылок при инициализации, ограничение доступных плагинов через поле tools режима (по умолчанию ['All']), требование использования имён модулей плагинов.
  • Обработчики Telegram: нормализация команд через PluginManager.get_plugin_commands(), запрет пробелов в именах команд, обработка фильтров сообщений без eval.
  • База данных: singleton Database с thread-local соединениями SQLite, WAL, busy_timeout, JSON-структура conversation_context.context как {"messages": [...]}, подсчёт message_count по сообщениям роли user, ограничение max_sessions с удалением старейшей сессии, вынос длительных вызовов OpenAI за пределы транзакций.
  • Тестирование: конфигурация pytest в pytest.ini (asyncio_mode = auto), расположение тестов в tests/ и bot/tests/, привязка тестовых файлов к конкретным компонентам системы.
  • Документация: AGENTS.md как активные инструкции (не история измений), приоритет кода над README при расхождениях, вынос исторических планов в отдельные датированные документы.

Документация Telegram-бота ChatGPT с интеграцией OpenAI/LLM-совместимых API. Основной файл README.md описывает архитектуру, развёртывание и конфигурацию через переменные окружения.

Ключевые компоненты системы:

  • LLMGateway — трёхуровневый маршрутизатор моделей (high / light_model / big_context) с настраиваемыми LLMGATEWAY_*_MODEL и базовым URL
  • OpenAIHelper — клиент для чата, vision, генерации изображений, TTS, транскрипции
  • ChatGPTTelegramBot — обработчик Telegram-обновлений с inline-режимом и сессионным управлением
  • PluginManager — система плагинов с namespace-инструментами, JSON-схемами и ~43 встроенными модулями
  • Database — SQLite с WAL, именованные сессии (MAX_SESSIONS с автоочисткой), бюджеты пользователей
  • Hindsight — опциональная долгосрочная память с автовызовом/автосохранением при удалении сессии
  • Agent runtime (agent_tools) — подагенты (до 5 параллельно, 10–50 раундов), планировщик задач (TASKS_TTL_SECONDS=2 дня), взаимодействие с пользователем
  • Skills runtime (skills) — локальные Codex-стиль папки SKILL.md с выполнением скриптов
  • MCP integration — подключение удалённых MCP-серверов как динамических инструментов модели

Режимы чата (bot/chat_modes.yml) — поведенческие профили с системным промптом, отображаемым именем и белым списком allowed_plugins. Валидация перекрёстных ссылок через chat_modes_registry.py; изменения требуют перезапуска.

Обязательные переменные: TELEGRAM_BOT_TOKEN, OPENAI_API_KEY.

  1. Создаёт временную директорию plugins через фикстуру tmp_path
