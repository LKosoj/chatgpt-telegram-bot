Поток данных и инициализация

Последовательность запуска

Точка входа — bot/__main__.py. Инициализация строго упорядочена:

PluginManager → Database → OpenAIHelper → ChatGPTTelegramBot
Компонент Ответственность
PluginManager Загрузка плагинов из bot/plugins/*.py, валидация дубликатов, неймспейсинг функций
Database Singleton SQLite с WAL, thread-local соединения, busy_timeout
OpenAIHelper Маршрутизация к LLM, управление диалогами, tool calls, мультимодальность
ChatGPTTelegramBot Polling Telegram, обработка обновлений, сессионное управление

Поток обработки запроса

  1. Telegram-polling — telegram_bot.py принимает обновление через базовый URL http://localhost:8081/bot (при TELEGRAM_LOCAL_MODE=true)
  2. Контекст запроса — RequestContext формирует неизменяемый контейнер с chat_id, user_id, message_id, session_id
  3. Ключ разговора — для групп используется chat_id, для личных чатов — user_id
  4. Режим чата — ChatModesRegistry загружает профили из bot/chat_modes.yml, валидирует tool-ссылки
  5. LLM-запрос — OpenAIHelper маршрутизирует через LLMGateway по трём уровням: high / light_model / big_context
  6. Tool calls — openai_tool_handler.py парсит ответ модели, валидирует аргументы по JSON Schema, маршрутизирует к плагинам
  7. Плагины — PluginManager разрешает имена через function_prefix, выполняет пакетно через asyncio.gather, инъектирует chat_id/user_id
  8. Результат — прямой ответ плагина (direct_result) либо возвращается пользователю напрямую, либо подаётся обратно в LLM-контекст

Плагиновая архитектура

Базовый класс и идентификация

Абстрактный класс Plugin в bot/plugins/plugin.py определяет обязательный интерфейс:

  • plugin_id — уникальный идентификатор плагина
  • function_prefix — неймспейс для всех функций плагина (предотвращает коллизии)
  • get_spec() — декларация инструментов в формате OpenAI Function Calling
  • execute() — асинхронное выполнение с инъекцией request_context

Загрузка и валидация

Параметр Назначение
PLUGINS CSV allow-list разрешённых плагинов
PLUGIN_STRICT_VALIDATION Строгая валидация: true → ValueError при дублирующихся именах функций между плагинами
PLUGIN_STORAGE_ROOT Корень хранилища плагинов, по умолчанию data/
PLUGIN_MENU_PAGE_SIZE 8 пунктов на страницу меню

Плагины загружаются из bot/plugins/*.py. Дубли в режиме мягкой валидации разрешаются через префикс (deepl.translate vs ddg_translate.translate).

Специальные плагины

Плагин Назначение Критические особенности
AgentToolsPlugin Подагенты, планировщик задач, доставка результатов До 5 параллельных подагентов, 10–50 раундов tool calls, TTL задач 2 дня, дедупликация доставки 60 сек
SkillsPlugin Локальные скрипты в стиле Codex SKILL.md с YAML frontmatter, entrypoints, аудит в skills_audit.jsonl, таймаут скриптов 120 сек
MCPServerPlugin Удалённые MCP-серверы HTTP/stdio транспорты, динамическая регистрация инструментов, JSON-RPC
HindsightMemoryPlugin Долговременная семантическая память Автовызов/автосохранение при удалении сессии, банки памяти по telegram-{user_id}

Режимы чата

Конфигурация в bot/chat_modes.yml. Каждый режим — поведенческий профиль:

Поле Описание
system_prompt Системный промпт, подаваемый в LLM
model_profile Маршруты LLMGateway: high/light/big_context/vision/image_generation/image_edit/transcription/tts
tools Белый список плагинов, по умолчанию ['All']
plugins_allowlist / plugins_denylist Явное разрешение/запрет плагинов
defer_direct_results Отложенная доставка результатов в агентном режиме
agent_limits Ограничения подагентов для этого режима
budget Лимиты токенов/времени

Валидация при старте: уникальность id, корректность ссылок на плагины, валидность regex. При STRICT_MODE=true ошибки блокируют запуск.

Управление сессиями и база данных

SQLite-архитектура

Database — thread-safe singleton с контекстным менеджером транзакций:

  • SQLITE_JOURNAL_MODE=WAL — write-ahead logging для конкурентности
  • SQLITE_BUSY_TIMEOUT_MS — ожидание при блокировке
  • Thread-local соединения — каждый поток получает свой sqlite3.Connection

Сессии

Параметр Значение по умолчанию Описание
MAX_SESSIONS 5 Максимум именованных сессий на пользователя
MAX_HISTORY_SIZE — Лимит сообщений в контексте
MAX_CONVERSATION_AGE_MINUTES — Автоочистка устаревших сессий

При превышении max_sessions — удаление старейшей сессии с опциональным финализатором Hindsight.

Структура контекста

{
  "messages": [
    {"role": "system", "content": "..."},
    {"role": "user", "content": "..."},
    {"role": "assistant", "content": "...", "tool_calls": [...]},
    {"role": "tool", "tool_call_id": "...", "content": "..."}
  ]
}

Поле message_count считается по сообщениям роли user. Длительные вызовы OpenAI выполняются вне транзакций.

LLM Gateway и маршрутизация

Трёхуровневая модель

Уровень Переменные Назначение
high LLMGATEWAY_HIGH_MODEL Сложные задачи, основной диалог
light_model LLMGATEWAY_LIGHT_MODEL Классификация, простые запросы
big_context LLMGATEWAY_BIG_CONTEXT_MODEL Длинные контексты, суммаризация

Мультимодальность

Возможность Флаги и модели
Vision ENABLE_VISION, VISION_MODEL, VISION_DETAIL
TTS ENABLE_TTS, TTS_MODEL, TTS_VOICE, TTS_SPEED
Транскрипция ENABLE_TRANSCRIPTION, TRANSCRIPTION_MODEL, WHISPER_MODEL
Генерация изображений ENABLE_IMAGE_GENERATION, IMAGE_GENERATION_MODEL, IMAGE_SIZE, IMAGE_QUALITY, IMAGE_STYLE
Редактирование изображений IMAGE_EDIT_MODEL

Клиент LLMGatewayClient

Асинхронный HTTP-клиент на httpx.AsyncClient:

  • Базовый URL обязателен (LLMGATEWAY_BASE_URL или OPENAI_API_BASE)
  • Таймауты: глобальный 120 сек, web_deep_research 1800 сек, web_research/image_edit 300 сек
  • Заголовок X-Title: tgBot для идентификации
  • Методы: web_search, web_read, web_research, web_deep_research, image_edit, audio_voices

Бюджеты и тарификация

Параметр Описание
BUDGET_PERIOD monthly / daily / weekly / total
USER_BUDGETS JSON-маппинг user_id → лимит, * = безлимит
GUEST_BUDGET Лимит для неавторизованных пользователей
TOKEN_PRICE Стоимость за токен
BUDGET_ALERT_THRESHOLD Порог предупреждения

Устаревшие переменные MONTHLY_USER_BUDGETS и MONTHLY_GUEST_BUDGET маппятся на новую схему с BUDGET_PERIOD=monthly и deprecation warning.

Развёртывание

Docker-образ

  • База: python:3.9-alpine
  • Системный пакет: ffmpeg для обработки медиа
  • Точка входа: python -m bot
  • Переменные сборки: PYTHONFAULTHANDLER=1, PYTHONUNBUFFERED=1, PYTHONDONTWRITEBYTECODE=1, PIP_DISABLE_PIP_VERSION_CHECK=on

Docker Compose

services:
  chatgpt-telegram-bot:
    build:
      context: .
      dockerfile: Dockerfile
    volumes:
      - .:/app          # горячая перезагрузка при разработке
    restart: unless-stopped

Обязательные переменные окружения

Переменная Описание
TELEGRAM_BOT_TOKEN Токен бота от @BotFather
OPENAI_API_KEY Ключ OpenAI API
LLMGATEWAY_BASE_URL или OPENAI_API_BASE Базовый URL LLM Gateway

Тестирование

Команда Покрытие
python -m pytest -q tests/ Основные тесты
python -m pytest -q bot/tests/ MCP-тесты
python -m py_compile bot/openai_helper.py bot/telegram_bot.py Синтаксическая проверка

Конфигурация: pytest.ini с asyncio_mode = auto. Тесты изолируют внешние зависимости через динамическую подмену sys.modules (tiktoken, pydub, markdown2, tenacity).
