Зависимости

Python-зависимости

Проект использует requirements.txt с жёстко зафиксированными версиями для воспроизводимости сборки. Ключевые группы:

Категория Библиотеки Назначение
Telegram python-telegram-bot, Telethon Интеграция с Bot API и MTProto
LLM/OpenAI openai, tiktoken, assemblyai API-вызовы, токенизация, транскрипция
HTTP-клиенты aiohttp, httpx, requests Асинхронные и синхронные запросы
Мультимедиа pydub, gtts, pytube, moviepy, ffmpeg-python Обработка аудио/видео, TTS, скачивание
Веб-скрапинг beautifulsoup4, lxml, trafilatura, readability-lxml, youtube-transcript-api, duckduckgo_search Извлечение контента, поиск, транскрипты
Веб-фреймворк fastapi, uvicorn Встроенный API (MCP, health checks)
ML/Аналитика numpy, pandas, scikit-learn, matplotlib, plotly Плагин codeinterpreter, визуализация
Тестирование pytest, pytest-asyncio Юнит-тесты с asyncio_mode = auto
Конфигурация PyYAML, jinja2, jsonschema, deep-translator Парсинг chat_modes.yml, валидация, i18n

Системные зависимости

Docker-образ python:3.9-alpine требует установки ffmpeg на этапе сборки — обязательный системный пакет для обработки аудио/видео плагинами (pydub, moviepy, транскрипция Whisper). Флаг --no-cache для apk минимизирует размер слоя.

Плагинные зависимости

Плагины загружаются динамически из bot/plugins/*.py с учётом allow-list PLUGINS (CSV). Каждый плагин может требовать собственных переменных окружения и Python-библиотек. Примеры:

  • MCPServerPlugin — требует mcp (опционально, pytest.importorskip в тестах)
  • CodeInterpreterPlugin — требует pandas, numpy, matplotlib, plotly (заглушаются в тестах)
  • AskYourPDFPlugin — PyPDF2, pdfminer, textract с fallback-цепочкой
  • SpotifyPlugin — spotipy с OAuth-флоу
  • VkusVillPlugin — удалённый MCP-сервер, транспорт HTTP/SSE

База данных

SQLite с WAL-режимом (SQLITE_JOURNAL_MODE=WAL). Драйвер встроенный (sqlite3), дополнительных зависимостей не требует. Для PostgreSQL-миграций присутствует asyncpg в requirements.txt, но основной путь — SQLite.

Управление зависимостями в runtime

  • Плагин PluginManager — валидация дубликатов функций через PLUGIN_STRICT_VALIDATION; namespace-инг через function_prefix для изоляции имён
  • Chat modes — валидация tool-ссылок при старте; несуществующие инструменты логируются как ERROR, при STRICT_MODE=true — блокируют запуск
  • MCP-серверы — валидация эндпоинта GET /tools на соответствие OpenAI Function Calling; динамическая регистрация инструментов с префиксом имени сервера
