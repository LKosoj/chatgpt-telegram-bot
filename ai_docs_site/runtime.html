Запуск и окружение

## Запуск и окружение

### Docker

Минимальная конфигурация для развёртывания бота в контейнере описана в `docker-compose.yml`. Сервис `chatgpt-telegram-bot` собирается из `Dockerfile` в текущей директории с контекстом `.`.

```yaml
services:
  chatgpt-telegram-bot:
    build:
      context: .
      dockerfile: Dockerfile
    volumes:
      - .:/app
    restart: unless-stopped
Параметр Назначение
volumes: .:/app Горячая перезагрузка кода при разработке — изменения на хосте немедленно видны в контейнере
restart: unless-stopped Автоматический перезапуск при сбоях или перезагрузке системы; ручная остановка (docker compose stop) предотвращает рестарт

Dockerfile (базовый образ python:3.9-alpine): - Устанавливает ffmpeg для обработки медиа - Отключает буферизацию и генерацию .pyc через переменные окружения - Точка входа: python -m bot

Переменная Значение Эффект
PYTHONFAULTHANDLER 1 Диагностика фатальных ошибок
PYTHONUNBUFFERED 1 Немедленный вывод логов
PYTHONDONTWRITEBYTECODE 1 Экономия места, отсутствие .pyc
PIP_DISABLE_PIP_VERSION_CHECK on Ускорение сборки

Переменные окружения

Единственный авторитетный источник runtime-параметров — файл .env (шаблон в .env.example). Код в bot/ имеет приоритет над документацией при расхождениях.

Обязательные

Переменная Описание
TELEGRAM_BOT_TOKEN Токен бота от @BotFather
OPENAI_API_KEY Ключ OpenAI или совместимого провайдера
LLMGATEWAY_BASE_URL (или OPENAI_API_BASE) Базовый URL LLM Gateway

Ключевые группы конфигурации

Telegram Core - ADMIN_USER_IDS — список администраторов через запятую - ALLOWED_TELEGRAM_USER_IDS — allow-list пользователей (* = все) - TELEGRAM_LOCAL_MODE — true для локального Bot API (localhost:8081) - TELEGRAM_BASE_URL — кастомный URL Bot API (проверяется только при local_mode=true)

LLM/Маршрутизация - LLMGATEWAY_HIGH_MODEL / LLMGATEWAY_LIGHT_MODEL / LLMGATEWAY_BIG_CONTEXT_MODEL — три уровня маршрутизации - OPENAI_BASE_URL — fallback для прямых вызовов OpenAI

Мультимодальность (все включаются через ENABLE_*) - VISION_MODEL + VISION_DETAIL — анализ изображений - TTS_MODEL + TTS_VOICE + TTS_SPEED — синтез речи - TRANSCRIPTION_MODEL / WHISPER_MODEL — распознавание аудио - IMAGE_GENERATION_MODEL + IMAGE_EDIT_MODEL + IMAGE_SIZE/IMAGE_QUALITY/IMAGE_STYLE — генерация и редактирование изображений

Сессии - MAX_HISTORY_SIZE — глубина истории сообщений - MAX_CONVERSATION_AGE_MINUTES — TTL диалога - MAX_SESSIONS — лимит именованных сессий на пользователя (по умолчанию 5, старейшая удаляется при превышении)

Бюджеты - BUDGET_PERIOD — monthly/daily/weekly/total - USER_BUDGETS — JSON-маппинг ID→лимит (* = безлимит) - GUEST_BUDGET — лимит для неавторизованных пользователей - TOKEN_PRICE + ценообразование по токенам/изображениям/TTS/vision/транскрипции

Плагины - PLUGINS — CSV allow-list загружаемых плагинов - PLUGIN_STRICT_VALIDATION — true вызывает ValueError при дублировании имён функций - PLUGIN_STORAGE_ROOT — корень хранилища плагинов (по умолчанию data/)

Безопасность Skills (требуют явного включения) - SKILLS_ALLOW_EXECUTION / SKILLS_ALLOW_SCRIPTS — опасно: скрипты выполняются с правами процесса - SKILLS_SCRIPT_ADMIN_USER_IDS — ограничение запуска скриптов администраторами - SKILLS_ALLOWED_OPERATORS — вайтлист операторов для codeinterpreter

MCP - MCP_SERVERS — JSON-конфиг подключённых серверов - DEFAULT_MCP_SERVERS — предустановленные в формате имя:url - MCP_SERVERS_ALLOWED_USERS — * для открытого доступа

Hindsight (долговременная память) - HINDSIGHT_ENABLED — мастер-переключатель - HINDSIGHT_BASE_URL + HINDSIGHT_API_TOKEN — эндпоинт и авторизация - HINDSIGHT_AUTO_RECALL / HINDSIGHT_AUTO_SAVE — автовызов при старте/удалении сессии

Database - DB_PATH — путь к SQLite (по умолчанию bot/user_data.db) - SQLITE_JOURNAL_MODE — WAL рекомендуется для конкурентного доступа - SQLITE_BUSY_TIMEOUT_MS — таймаут ожидания блокировки

Устаревшие переменные

Старое Новое Поведение
MONTHLY_USER_BUDGETS USER_BUDGETS + BUDGET_PERIOD=monthly fallback с DeprecationWarning
MONTHLY_GUEST_BUDGET GUEST_BUDGET + BUDGET_PERIOD=monthly fallback с DeprecationWarning

Инициализация runtime

Точка входа: bot/__main__.py. Последовательность инициализации строго фиксирована:

PluginManager → Database → OpenAIHelper → ChatGPTTelegramBot

Критические проверки при старте: 1. Валидация TELEGRAM_BOT_TOKEN и OPENAI_API_KEY — отсутствие вызывает немедленный ValueError 2. Валидация TELEGRAM_LOCAL_MODE и TELEGRAM_BASE_URL — невалидный URL или булево блокируют запуск до создания экземпляра бота 3. ChatModesRegistry валидирует bot/chat_modes.yml: уникальность id, корректность ссылок на плагины, валидность regex; при STRICT_MODE=true ошибки блокируют старт

Telegram-polling запускается в bot/telegram_bot.py с базовым URL http://localhost:8081/bot при локальном режиме.

Тестирование

# Полный набор
python -m pytest -q tests/
python -m pytest -q bot/tests/

# Синтаксическая проверка ключевых модулей
python -m py_compile bot/openai_helper.py bot/telegram_bot.py

Конфигурация pytest (pytest.ini или [pytest] в pyproject.toml): - asyncio_mode = auto — автоматическое обнаружение асинхронных тестов

Режимы чата

Декларируются в bot/chat_modes.yml, валидируются через chat_modes_registry.py. Изменения требуют перезапуска бота.

Структура режима: - id — уникальный идентификатор - system_prompt — поведенческий профиль - model_profile — маршруты LLMGateway (high/light/big_context/vision/image_generation/image_edit/transcription/tts) - plugins_allowlist / plugins_denylist — фильтрация инструментов (["All"] по умолчанию) - tools — ограничение доступных плагинов (имена модулей) - feature-флаги: vision, audio, web_search, deep_research, agent_runtime, hindsight, skills, mcp - agent_limits, tts_voice, response_format

Плагиновая система

Базовый класс: bot/plugins/plugin.py

Механизм Деталь
Идентификация plugin_id + function_prefix для неймспейсинга
Загрузка bot/plugins/*.py с учётом allow-list PLUGINS
Валидация дубликатов PLUGIN_STRICT_VALIDATION — строгая (ValueError) или мягкая (последний wins)
Выполнение asyncio.gather для параллельных tool calls
Инъекция контекста Автоматическая подстановка chat_id/user_id/message_id через RequestContext
Прямой ответ direct_result в плагине обходит повторный вызов модели

Команды плагинов нормализуются через PluginManager.get_plugin_commands() — пробелы в именах запрещены.
